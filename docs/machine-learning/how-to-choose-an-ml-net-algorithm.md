## <a name="linear-algorithms"></a><span data-ttu-id="29da1-124">线性算法</span><span class="sxs-lookup"><span data-stu-id="29da1-124">Linear algorithms</span></span>

<span data-ttu-id="29da1-125">线性算法生成一个模型，该模型根据输入数据和一组**权重**的线性组合计算**分数**。</span><span class="sxs-lookup"><span data-stu-id="29da1-125">Linear algorithms produce a model that calculates **scores** from a linear combination of the input data and a set of **weights**.</span></span> <span data-ttu-id="29da1-126">权重是训练期间估算的模型参数。</span><span class="sxs-lookup"><span data-stu-id="29da1-126">The weights are parameters of the model estimated during training.</span></span>

<span data-ttu-id="29da1-132">线性算法对训练数据进行多次传递。</span><span class="sxs-lookup"><span data-stu-id="29da1-132">Linear algorithms make multiple passes over the training data.</span></span> <span data-ttu-id="29da1-133">如果数据集适用于内存，则在追加训练程序之前向 ML.NET 管道添加[缓存检查点](xref:Microsoft.ML.LearningPipelineExtensions.AppendCacheCheckpoint*)将使训练运行速度加快。</span><span class="sxs-lookup"><span data-stu-id="29da1-133">If your dataset fits into memory, then adding a [cache checkpoint](xref:Microsoft.ML.LearningPipelineExtensions.AppendCacheCheckpoint*) to your ML.NET pipeline before appending the trainer, will make the training run faster.</span></span>

## <a name="decision-tree-algorithms"></a><span data-ttu-id="29da1-150">决策树算法</span><span class="sxs-lookup"><span data-stu-id="29da1-150">Decision tree algorithms</span></span>

<span data-ttu-id="29da1-151">决策树算法创建包含一系列决策的模型：实际上是包含数据值的流程图。</span><span class="sxs-lookup"><span data-stu-id="29da1-151">Decision tree algorithms create a model that contains a series of decisions: effectively a flow chart through the data values.</span></span>

<span data-ttu-id="29da1-152">特征不需要线性可分即可使用此类算法。</span><span class="sxs-lookup"><span data-stu-id="29da1-152">Features do not need to be linearly separable to use this type of algorithm.</span></span> <span data-ttu-id="29da1-153">特征无需规范化，因为特征向量中的各个值在决策过程中独立使用。</span><span class="sxs-lookup"><span data-stu-id="29da1-153">And features do not need to be normalized, because the individual values in the feature vector are used independently in the decision process.</span></span>
