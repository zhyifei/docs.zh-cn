# <a name="expression-of-type-typename1-can-never-be-of-type-typename2"></a><span data-ttu-id="bba73-102">类型的表达式\<typename1 > 可以永远不能为类型\<typename2 ></span><span class="sxs-lookup"><span data-stu-id="bba73-102">Expression of type '\<typename1>' can never be of type '\<typename2>'</span></span>
<span data-ttu-id="bba73-103">`TypeOf`...`Is` 表达式测试它不能包含的数据类型的对象引用变量。</span><span class="sxs-lookup"><span data-stu-id="bba73-103">A `TypeOf`...`Is` expression tests an object reference variable to a data type that it cannot hold.</span></span>  
  
 <span data-ttu-id="bba73-104">在某些情况下，例如，如果两个类之间没有任何继承关系，编译器可以确定 `TypeOf`...`Is` 测试只能失败。</span><span class="sxs-lookup"><span data-stu-id="bba73-104">In some cases the compiler can determine that a `TypeOf`...`Is` test can only fail, for example if there is no inheritance relationship between two classes.</span></span>  
  
 <span data-ttu-id="bba73-105">以下代码会生成此错误。</span><span class="sxs-lookup"><span data-stu-id="bba73-105">The following code can generate this error.</span></span>  
  
 `Dim refVar as System.Windows.Forms.Form`  
  
 `If TypeOf refVar Is System.Array`  
  
 `End If`  
  
 <span data-ttu-id="bba73-106">因为 <xref:System.Windows.Forms.Form> 和 <xref:System.Array> 是完全不相关的类型，编译器可以确定 `TypeOf`...`Is` 表达式为 `False` 的任何值返回 `refVar`。</span><span class="sxs-lookup"><span data-stu-id="bba73-106">Because <xref:System.Windows.Forms.Form> and <xref:System.Array> are totally unrelated types, the compiler can determine that the `TypeOf`...`Is` expression returns `False` for any value of `refVar`.</span></span>  
  
 <span data-ttu-id="bba73-107">**错误 ID:** BC31430</span><span class="sxs-lookup"><span data-stu-id="bba73-107">**Error ID:** BC31430</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="bba73-108">更正此错误</span><span class="sxs-lookup"><span data-stu-id="bba73-108">To correct this error</span></span>
