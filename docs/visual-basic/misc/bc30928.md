# <a name="base-class-39ltbaseclassname1gt39-specified-for-class-39ltpartialclassnamegt39-cannot-be-different-from-the-base-class-39ltbaseclassname2gt39-of-one-of-its-other-partial-types"></a>基类&#39; &lt;baseclassname1&gt; &#39;为类指定&#39; &lt;partialclassname&gt; &#39;不能从基类不同&#39; &lt;baseclassname2&gt; &#39;其其他分部类型之一
在两个或更多分部声明中定义了某个类，这些声明包含多个指定多个基类的 [Inherits Statement](../../visual-basic/language-reference/statements/inherits-statement.md) 。  
  
 在多个分部声明间划分类的定义时，编译器会将该类视为它的所有分部声明的联合。 这不仅适用于成员，还适用于实现、继承和访问级别。  
  
 一个类可以实现多个接口，但它无法从多个基类中继承。 因此，所有 `Inherits` 语句必须指定相同的基类。  
  
 **错误 ID：** BC30928  
  
## <a name="to-correct-this-error"></a>更正此错误  
  
-   确定哪种类应成为分部类的基类，并从其分部声明中删除任何指定其他基类的 `Inherits` 语句。
