# <a name="interfaces-visual-basic"></a><span data-ttu-id="0759b-102">接口 (Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="0759b-102">Interfaces (Visual Basic)</span></span>
<span data-ttu-id="0759b-103">接口定义了类可以实现的属性、方法和事件。</span><span class="sxs-lookup"><span data-stu-id="0759b-103">*Interfaces* define the properties, methods, and events that classes can implement.</span></span> <span data-ttu-id="0759b-104">接口允许将功能定义为一些紧密相关的属性、方法和事件的小组；这样就减少了兼容性问题，因为可以在不损害现有代码的情况下开发接口的增强型实现。</span><span class="sxs-lookup"><span data-stu-id="0759b-104">Interfaces allow you to define features as small groups of closely related properties, methods, and events; this reduces compatibility problems because you can develop enhanced implementations for your interfaces without jeopardizing existing code.</span></span> <span data-ttu-id="0759b-105">在任何时候都可以通过开发附加接口和实现来添加新的功能。</span><span class="sxs-lookup"><span data-stu-id="0759b-105">You can add new features at any time by developing additional interfaces and implementations.</span></span>  
  
 <span data-ttu-id="0759b-106">以下是为何要使用接口继承而不用类继承的一些其他原因：</span><span class="sxs-lookup"><span data-stu-id="0759b-106">There are several other reasons why you might want to use interfaces instead of class inheritance:</span></span>  
  
-   <span data-ttu-id="0759b-107">在应用程序要求很多可能不相关的对象类型以提供某种功能的情况下，接口的适用性更强。</span><span class="sxs-lookup"><span data-stu-id="0759b-107">Interfaces are better suited to situations in which your applications require many possibly unrelated object types to provide certain functionality.</span></span>  
  
-   <span data-ttu-id="0759b-108">接口比基类更灵活，因为可以定义单个实现来实现多个接口。</span><span class="sxs-lookup"><span data-stu-id="0759b-108">Interfaces are more flexible than base classes because you can define a single implementation that can implement multiple interfaces.</span></span>  
  
-   <span data-ttu-id="0759b-109">在无需从基类继承实现的情况下，接口更好。</span><span class="sxs-lookup"><span data-stu-id="0759b-109">Interfaces are better in situations in which you do not have to inherit implementation from a base class.</span></span>  
  
-   <span data-ttu-id="0759b-110">在无法使用类继承的情况下接口非常有用。</span><span class="sxs-lookup"><span data-stu-id="0759b-110">Interfaces are useful when you cannot use class inheritance.</span></span> <span data-ttu-id="0759b-111">例如，结构无法从类继承，但它们可以实现接口。</span><span class="sxs-lookup"><span data-stu-id="0759b-111">For example, structures cannot inherit from classes, but they can implement interfaces.</span></span>  
  
## <a name="declaring-interfaces"></a><span data-ttu-id="0759b-112">声明接口</span><span class="sxs-lookup"><span data-stu-id="0759b-112">Declaring Interfaces</span></span>
