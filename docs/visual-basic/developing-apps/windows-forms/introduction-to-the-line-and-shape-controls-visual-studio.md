-   <span data-ttu-id="2291d-114">Line 和 Shape 控件是可选择在运行时，如提供事件<xref:Microsoft.VisualBasic.PowerPacks.Shape.Click>和<xref:Microsoft.VisualBasic.PowerPacks.Shape.OnDoubleClick%2A>。</span><span class="sxs-lookup"><span data-stu-id="2291d-114">Line and Shape controls are selectable at run time, providing events such as <xref:Microsoft.VisualBasic.PowerPacks.Shape.Click> and <xref:Microsoft.VisualBasic.PowerPacks.Shape.OnDoubleClick%2A>.</span></span> <span data-ttu-id="2291d-115">图形方法的输出是不可选择的并且不提供事件。</span><span class="sxs-lookup"><span data-stu-id="2291d-115">The outputs of graphics methods are not selectable and do not provide events.</span></span>  
  
-   <span data-ttu-id="2291d-116">Line 和 Shape 控件提供<xref:Microsoft.VisualBasic.PowerPacks.Shape.BringToFront%2A>和<xref:Microsoft.VisualBasic.PowerPacks.Shape.SendToBack%2A>使您可以在设计时和在运行时控制其 z 顺序的方法。</span><span class="sxs-lookup"><span data-stu-id="2291d-116">Line and Shape controls provide <xref:Microsoft.VisualBasic.PowerPacks.Shape.BringToFront%2A> and <xref:Microsoft.VisualBasic.PowerPacks.Shape.SendToBack%2A> methods that enable you to control their z-order at design time and at run time.</span></span> <span data-ttu-id="2291d-117">可以仅通过更改其顺序在运行时执行的控制 z 顺序的图形方法。</span><span class="sxs-lookup"><span data-stu-id="2291d-117">The z-order of graphics methods can be controlled only by changing their order of execution at run time.</span></span>  
  
-   <span data-ttu-id="2291d-118">Line 和 Shape 控件是无窗口控件;它们没有窗口句柄，因此使用少的系统资源。</span><span class="sxs-lookup"><span data-stu-id="2291d-118">Line and Shape controls are windowless controls; they have no window handles and therefore use less system resources.</span></span>
