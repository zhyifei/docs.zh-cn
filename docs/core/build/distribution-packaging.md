# <a name="net-core-distribution-packaging"></a><span data-ttu-id="e31de-103">.NET Core 分发打包</span><span class="sxs-lookup"><span data-stu-id="e31de-103">.NET Core distribution packaging</span></span>

## <a name="disk-layout"></a><span data-ttu-id="e31de-106">磁盘布局</span><span class="sxs-lookup"><span data-stu-id="e31de-106">Disk layout</span></span>

<span data-ttu-id="e31de-107">安装时，.NET Core 包含一些组件，这些组件在文件系统中排列如下：</span><span class="sxs-lookup"><span data-stu-id="e31de-107">When installed, .NET Core consists of several components that are layed out as follows in the filesystem:</span></span>

```
.
├── dotnet                       (1)
├── LICENSE.txt                  (8)
├── ThirdPartyNotices.txt        (8)
├── host
│   └── fxr
│       └── <fxr version>        (2)
├── sdk
│   ├── <sdk version>            (3)
│   └── NuGetFallbackFolder      (4)
└── shared
    ├── Microsoft.NETCore.App
    │   └── <runtime version>    (5)
    └── Microsoft.AspNetCore.App
        └── <aspnetcore version> (6)
    └── Microsoft.AspNetCore.All
        └── <aspnetcore version> (7)
/
├─usr/share/man/man1
│       └── dotnet.1.gz          (9)
└─usr/bin
        └── dotnet               (10)
```

- <span data-ttu-id="e31de-108">(1) **dotnet** 主机（也称为“muxer”）有两个不同角色：激活运行时以启动应用程序，及激活 SDK 以向其分派命令。</span><span class="sxs-lookup"><span data-stu-id="e31de-108">(1) **dotnet** The host (also known as the "muxer") has two distinct roles: activate a runtime to launch an application, and activate an SDK to dispatch commands to it.</span></span> <span data-ttu-id="e31de-109">主机是本机可执行文件 (`dotnet.exe`)。</span><span class="sxs-lookup"><span data-stu-id="e31de-109">The host is a native executable (`dotnet.exe`).</span></span>

<span data-ttu-id="e31de-110">主机只有一个，不过大部分的其他组件都在带有版本的目录中（2、3、5 和 6）。</span><span class="sxs-lookup"><span data-stu-id="e31de-110">While there is a single host, most of the other components are in versioned directories (2,3,5,6).</span></span> <span data-ttu-id="e31de-111">也就是说，因为它们是并行安装的，所以系统上可以出现多个版本。</span><span class="sxs-lookup"><span data-stu-id="e31de-111">These means multiple versions can be present on the system since they are installed side-by-side.</span></span>

- <span data-ttu-id="e31de-112">(2) **host/fxr/\<fxr version>** 包含了主机所使用的框架解析逻辑。</span><span class="sxs-lookup"><span data-stu-id="e31de-112">(2) **host/fxr/\<fxr version>** contains the framework resolution logic used by the host.</span></span> <span data-ttu-id="e31de-113">主机采用已安装的最新 hostfxr。</span><span class="sxs-lookup"><span data-stu-id="e31de-113">The host uses the latest hostfxr that is installed.</span></span> <span data-ttu-id="e31de-114">在执行 .NET Core 应用程序时，hostfxr 负责选择合适的运行时。</span><span class="sxs-lookup"><span data-stu-id="e31de-114">The hostfxr is responsible for selecting the appropriate runtime when executing a .NET Core application.</span></span> <span data-ttu-id="e31de-115">例如，.NET Core 2.0.0 的应用程序会使用 2.0.5 运行时（如果可用）。</span><span class="sxs-lookup"><span data-stu-id="e31de-115">For example, an application built for .NET Core 2.0.0 will use the 2.0.5 runtime when it is available.</span></span> <span data-ttu-id="e31de-116">同样，hostfxr 在开发期间也会选择适当的 SDK。</span><span class="sxs-lookup"><span data-stu-id="e31de-116">Similarly, hostfxr selects the appropriate SDK during development.</span></span>

- <span data-ttu-id="e31de-117">(3) **sdk/\<sdk version>** SDK（也称为“工具”）是一组托管工具，可用于编写和生成 .NET Core 库和应用程序。</span><span class="sxs-lookup"><span data-stu-id="e31de-117">(3) **sdk/\<sdk version>** The SDK (also known as "the tooling") is a set of managed tools that can be used to write and build .NET Core libraries and applications.</span></span> <span data-ttu-id="e31de-118">SDK 包括 CLI、Roslyn 编译器、MSBuild 及相关生成任务和目标、NuGet、新项目模板等。</span><span class="sxs-lookup"><span data-stu-id="e31de-118">The SDK includes the CLI, the Roslyn compiler, MSBuild, and associated build tasks and targets, NuGet, new project templates, etc.</span></span>

- <span data-ttu-id="e31de-119">(4) **sdk/NuGetFallbackFolder** 包含 `dotnet restore` 步骤中 SDK 所使用的 NuGet 包的缓存。</span><span class="sxs-lookup"><span data-stu-id="e31de-119">(4) **sdk/NuGetFallbackFolder** contains a cache of NuGet packages used by an SDK during the `dotnet restore` step.</span></span>

<span data-ttu-id="e31de-120">“共享”文件夹包含框架。</span><span class="sxs-lookup"><span data-stu-id="e31de-120">The **shared** folder contains frameworks.</span></span> <span data-ttu-id="e31de-121">共享框架提供一组位于中心位置的库，从而让不同的应用程序使用。</span><span class="sxs-lookup"><span data-stu-id="e31de-121">A shared framework provides a set of libraries at a central location so they can be used by different applications.</span></span>

- <span data-ttu-id="e31de-122">(5) **shared/Microsoft.NETCore.App/\<runtime version>** 此框架包含.NET Core 运行时和支持托管库。</span><span class="sxs-lookup"><span data-stu-id="e31de-122">(5) **shared/Microsoft.NETCore.App/\<runtime version>** This framework contains the .NET Core runtime and supporting managed libraries.</span></span>

- <span data-ttu-id="e31de-123">(6,7) **shared/Microsoft.AspNetCore.{App,All}/\<aspnetcore version>** 包含 ASP.NET Core 库。</span><span class="sxs-lookup"><span data-stu-id="e31de-123">(6,7) **shared/Microsoft.AspNetCore.{App,All}/\<aspnetcore version>** contains the ASP.NET Core libraries.</span></span> <span data-ttu-id="e31de-124">已开发且支持 `Microsoft.AspNetCore.App` 下的库（作为 .NET Core 项目的一部分）。</span><span class="sxs-lookup"><span data-stu-id="e31de-124">The libraries under `Microsoft.AspNetCore.App` are developed and supported as part of the .NET Core project.</span></span> <span data-ttu-id="e31de-125">`Microsoft.AspNetCore.All` 下的库是一个超集，其中还包含第三方库。</span><span class="sxs-lookup"><span data-stu-id="e31de-125">The libraries under `Microsoft.AspNetCore.All` are a superset which also contains 3rd party libraries.</span></span>

- <span data-ttu-id="e31de-126">(8) **LICENSE.txt,ThirdPartyNotices.txt** 是 .NET Core 许可证和 .NET Core 中使用的第三方库的许可证。</span><span class="sxs-lookup"><span data-stu-id="e31de-126">(8) **LICENSE.txt,ThirdPartyNotices.txt** are the .NET Core license and licenses of third-party libraries used in .NET Core.</span></span>
