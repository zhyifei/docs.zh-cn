## <a name="what-is-a-parallel-query"></a><span data-ttu-id="c9131-103">什么是并行查询？</span><span class="sxs-lookup"><span data-stu-id="c9131-103">What is a Parallel Query?</span></span>  
 <span data-ttu-id="c9131-104">[!INCLUDE[net_v35_long](../../../includes/net-v35-long-md.md)] 中引入了语言集成查询 (LINQ)。</span><span class="sxs-lookup"><span data-stu-id="c9131-104">Language-Integrated Query (LINQ) was introduced in the [!INCLUDE[net_v35_long](../../../includes/net-v35-long-md.md)].</span></span>  <span data-ttu-id="c9131-105">它具有统一的模型，以类型安全方式查询任何 <xref:System.Collections.IEnumerable?displayProperty=nameWithType> 或 <xref:System.Collections.Generic.IEnumerable%601?displayProperty=nameWithType> 数据源。</span><span class="sxs-lookup"><span data-stu-id="c9131-105">It features a unified model for querying any <xref:System.Collections.IEnumerable?displayProperty=nameWithType> or <xref:System.Collections.Generic.IEnumerable%601?displayProperty=nameWithType> data source in a type-safe manner.</span></span> <span data-ttu-id="c9131-106">LINQ to Objects 是针对内存中集合（如 <xref:System.Collections.Generic.List%601> 和数组）运行的 LINQ 查询的名称。</span><span class="sxs-lookup"><span data-stu-id="c9131-106">LINQ to Objects is the name for LINQ queries that are run against in-memory collections such as <xref:System.Collections.Generic.List%601> and arrays.</span></span> <span data-ttu-id="c9131-107">本文假定你对 LINQ 有基本的了解。</span><span class="sxs-lookup"><span data-stu-id="c9131-107">This article assumes that you have a basic understanding of LINQ.</span></span> <span data-ttu-id="c9131-108">有关详细信息，请参阅[语言集成查询 (LINQ) - C#](../../csharp/programming-guide/concepts/linq/index.md) 或[语言集成查询 (LINQ) - Visual Basic](../../visual-basic/programming-guide/concepts/linq/index.md)。</span><span class="sxs-lookup"><span data-stu-id="c9131-108">For more information, see [Language-Integrated Query (LINQ) - C#](../../csharp/programming-guide/concepts/linq/index.md) or [Language-Integrated Query (LINQ) - Visual Basic](../../visual-basic/programming-guide/concepts/linq/index.md).</span></span>  
  
 <span data-ttu-id="c9131-109">并行 LINQ (PLINQ) 是 LINQ 模式的并行实现。</span><span class="sxs-lookup"><span data-stu-id="c9131-109">Parallel LINQ (PLINQ) is a parallel implementation of the LINQ pattern.</span></span> <span data-ttu-id="c9131-110">一个 PLINQ 查询的许多方面都类似于非并行的 LINQ to Objects 查询。</span><span class="sxs-lookup"><span data-stu-id="c9131-110">A PLINQ query in many ways resembles a non-parallel LINQ to Objects query.</span></span> <span data-ttu-id="c9131-111">与顺序 [!INCLUDE[vbteclinq](../../../includes/vbteclinq-md.md)] 查询一样，PLINQ 查询对任何内存中 <xref:System.Collections.IEnumerable> 或 <xref:System.Collections.Generic.IEnumerable%601> 数据源执行操作，并且推迟了执行，即在枚举查询前不会开始执行。</span><span class="sxs-lookup"><span data-stu-id="c9131-111">PLINQ queries, just like sequential [!INCLUDE[vbteclinq](../../../includes/vbteclinq-md.md)] queries, operate on any in-memory <xref:System.Collections.IEnumerable> or <xref:System.Collections.Generic.IEnumerable%601> data source, and have deferred execution, which means they do not begin executing until the query is enumerated.</span></span> <span data-ttu-id="c9131-112">主要区别在于，PLINQ 会尝试充分利用系统上的所有处理器。</span><span class="sxs-lookup"><span data-stu-id="c9131-112">The primary difference is that PLINQ attempts to make full use of all the processors on the system.</span></span> <span data-ttu-id="c9131-113">方法是将数据源分区成片段，然后在多个处理器上针对单独工作线程上的每个片段执行并行查询。</span><span class="sxs-lookup"><span data-stu-id="c9131-113">It does this by partitioning the data source into segments, and then executing the query on each segment on separate worker threads in parallel on multiple processors.</span></span> <span data-ttu-id="c9131-114">在许多情况下，并行执行意味着查询运行速度显著提高。</span><span class="sxs-lookup"><span data-stu-id="c9131-114">In many cases, parallel execution means that the query runs significantly faster.</span></span>  
  
 <span data-ttu-id="c9131-115">通过并行执行，通常只需向数据源添加 <xref:System.Linq.ParallelEnumerable.AsParallel%2A> 查询操作，PLINQ 即可显著提升性能（与某些类型查询的旧代码相比）。</span><span class="sxs-lookup"><span data-stu-id="c9131-115">Through parallel execution, PLINQ can achieve significant performance improvements over legacy code for certain kinds of queries, often just by adding the <xref:System.Linq.ParallelEnumerable.AsParallel%2A> query operation to the data source.</span></span> <span data-ttu-id="c9131-116">但是，并行可能会引入其自身的复杂性，因此并非所有的查询操作的运行速度在 PLINQ 中都更快。</span><span class="sxs-lookup"><span data-stu-id="c9131-116">However, parallelism can introduce its own complexities, and not all query operations run faster in PLINQ.</span></span> <span data-ttu-id="c9131-117">事实上，并行实际上会降低某些查询的速度。</span><span class="sxs-lookup"><span data-stu-id="c9131-117">In fact, parallelization actually slows down certain queries.</span></span> <span data-ttu-id="c9131-118">因此，应了解排序等问题将如何对并行查询产生影响。</span><span class="sxs-lookup"><span data-stu-id="c9131-118">Therefore, you should understand how issues such as ordering affect parallel queries.</span></span> <span data-ttu-id="c9131-119">有关详细信息，请参阅[了解 PLINQ 中的加速](../../../docs/standard/parallel-programming/understanding-speedup-in-plinq.md)。</span><span class="sxs-lookup"><span data-stu-id="c9131-119">For more information, see [Understanding Speedup in PLINQ](../../../docs/standard/parallel-programming/understanding-speedup-in-plinq.md).</span></span>
