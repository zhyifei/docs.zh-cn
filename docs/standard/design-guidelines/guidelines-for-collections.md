## <a name="implementing-custom-collections"></a><span data-ttu-id="2b4d9-169">实现自定义集合</span><span class="sxs-lookup"><span data-stu-id="2b4d9-169">Implementing Custom Collections</span></span>  
 <span data-ttu-id="2b4d9-170">**✓ 考虑**在设计新集合时，从 `Collection<T>`、`ReadOnlyCollection<T>` 或 `KeyedCollection<TKey,TItem>` 继承。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-170">**✓ CONSIDER** inheriting from `Collection<T>`, `ReadOnlyCollection<T>`, or `KeyedCollection<TKey,TItem>` when designing new collections.</span></span>  
  
 <span data-ttu-id="2b4d9-171">**✓ 务必**在设计新集合时，实现 `IEnumerable<T>`。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-171">**✓ DO** implement `IEnumerable<T>` when designing new collections.</span></span> <span data-ttu-id="2b4d9-172">当有需要时，考虑实现 `ICollection<T>` 甚至 `IList<T>`。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-172">Consider implementing `ICollection<T>` or even `IList<T>` where it makes sense.</span></span>  
  
 <span data-ttu-id="2b4d9-173">在实现此类自定义集合时，请尽可能遵循 `Collection<T>` 和 `ReadOnlyCollection<T>` 建立的 API 模式。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-173">When implementing such custom collection, follow the API pattern established by `Collection<T>` and `ReadOnlyCollection<T>` as closely as possible.</span></span> <span data-ttu-id="2b4d9-174">也就是说，显式实现相同的成员，以及命名参数（与这两个集合命名它们的方式类似）等。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-174">That is, implement the same members explicitly, name the parameters like these two collections name them, and so on.</span></span>  
  
 <span data-ttu-id="2b4d9-175">**✓ 考虑**实现非泛型集合接口（`IList` 和 `ICollection`），如果该集合经常被传递给以这些接口为输入的 API。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-175">**✓ CONSIDER** implementing nongeneric collection interfaces (`IList` and `ICollection`) if the collection will often be passed to APIs taking these interfaces as input.</span></span>  
  
 <span data-ttu-id="2b4d9-176">**X 避免** 在具有与集合概念无关的复杂 API 的类型上实现集合接口。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-176">**X AVOID** implementing collection interfaces on types with complex APIs unrelated to the concept of a collection.</span></span>  
  
 <span data-ttu-id="2b4d9-177">**X 切忌**从非泛型基本集合继承，如 `CollectionBase`。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-177">**X DO NOT** inherit from nongeneric base collections such as `CollectionBase`.</span></span> <span data-ttu-id="2b4d9-178">相反，使用 `Collection<T>`、`ReadOnlyCollection<T>` 和 `KeyedCollection<TKey,TItem>`。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-178">Use `Collection<T>`, `ReadOnlyCollection<T>`, and `KeyedCollection<TKey,TItem>` instead.</span></span>  
  
### <a name="naming-custom-collections"></a><span data-ttu-id="2b4d9-179">命名自定义集合</span><span class="sxs-lookup"><span data-stu-id="2b4d9-179">Naming Custom Collections</span></span>  
 <span data-ttu-id="2b4d9-180">集合 (类型实现`IEnumerable`) 创建主要有两个原因：（1） 若要创建新的数据结构与特定于结构的操作，通常不同的性能特征与现有数据结构 (例如， <xref:System.Collections.Generic.List%601>， <xref:System.Collections.Generic.LinkedList%601>， <xref:System.Collections.Generic.Stack%601>)，和 （2） 若要创建的专用的集合包含一组特定的项 (例如， <xref:System.Collections.Specialized.StringCollection>)。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-180">Collections (types that implement `IEnumerable`) are created mainly for two reasons: (1) to create a new data structure with structure-specific operations and often different performance characteristics than existing data structures (e.g.,  <xref:System.Collections.Generic.List%601>, <xref:System.Collections.Generic.LinkedList%601>, <xref:System.Collections.Generic.Stack%601>), and (2) to create a specialized collection for holding a specific set of items (e.g.,  <xref:System.Collections.Specialized.StringCollection>).</span></span> <span data-ttu-id="2b4d9-181">数据结构最常用于应用程序和库的内部实现。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-181">Data structures are most often used in the internal implementation of applications and libraries.</span></span> <span data-ttu-id="2b4d9-182">专用集合主要在 API 中公开（作为属性和参数类型）。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-182">Specialized collections are mainly to be exposed in APIs (as property and parameter types).</span></span>  
  
 <span data-ttu-id="2b4d9-183">**✓ 务必**在实现 `IDictionary` 或 `IDictionary<TKey,TValue>` 的抽象名称中使用“Dictionary”后缀。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-183">**✓ DO** use the "Dictionary" suffix in names of abstractions implementing `IDictionary` or `IDictionary<TKey,TValue>`.</span></span>  
  
 <span data-ttu-id="2b4d9-184">**✓ 务必**在实现 `IEnumerable`（或其任意后代）并表示项列表的类型名称中使用“Collection”后缀。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-184">**✓ DO** use the "Collection" suffix in names of types implementing `IEnumerable` (or any of its descendants) and representing a list of items.</span></span>  
  
 <span data-ttu-id="2b4d9-185">**✓ 务必**为自定义数据结构使用适当的数据结构名称。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-185">**✓ DO** use the appropriate data structure name for custom data structures.</span></span>  
  
 <span data-ttu-id="2b4d9-186">**X 避免**在集合抽象名称中使用任何暗示特定实现的后缀，例如“LinkedList”或“Hashtable”。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-186">**X AVOID** using any suffixes implying particular implementation, such as "LinkedList" or "Hashtable," in names of collection abstractions.</span></span>  
  
 <span data-ttu-id="2b4d9-187">**✓ 考虑**在集合名称前加上项类型的名称。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-187">**✓ CONSIDER** prefixing collection names with the name of the item type.</span></span> <span data-ttu-id="2b4d9-188">例如，存储类型为 `Address` 的项的集合（实现 `IEnumerable<Address>`）应该命名为 `AddressCollection`。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-188">For example, a collection storing items of type `Address` (implementing `IEnumerable<Address>`) should be named `AddressCollection`.</span></span> <span data-ttu-id="2b4d9-189">如果项类型是接口，则可以省略项类型的 “I” 前缀。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-189">If the item type is an interface, the "I" prefix of the item type can be omitted.</span></span> <span data-ttu-id="2b4d9-190">因此，<xref:System.IDisposable> 项的集合可以称为 `DisposableCollection`。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-190">Thus, a collection of <xref:System.IDisposable> items can be called `DisposableCollection`.</span></span>  
  
 <span data-ttu-id="2b4d9-191">**✓ 考虑**如果框架中可能已添加或已存在相应的可写集合，则在只读集合的名称中使用“ReadOnly”前缀。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-191">**✓ CONSIDER** using the "ReadOnly" prefix in names of read-only collections if a corresponding writeable collection might be added or already exists in the framework.</span></span>  
  
 <span data-ttu-id="2b4d9-192">例如，只读字符串集合应称为 `ReadOnlyStringCollection`。</span><span class="sxs-lookup"><span data-stu-id="2b4d9-192">For example, a read-only collection of strings should be called `ReadOnlyStringCollection`.</span></span>
