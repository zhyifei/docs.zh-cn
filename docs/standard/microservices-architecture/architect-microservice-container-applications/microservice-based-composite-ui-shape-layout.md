# <a name="creating-composite-ui-based-on-microservices-including-visual-ui-shape-and-layout-generated-by-multiple-microservices"></a><span data-ttu-id="398ad-104">根据微服务创建复合 UI，包括多个微服务生成的可视 UI 形状和布局</span><span class="sxs-lookup"><span data-stu-id="398ad-104">Creating composite UI based on microservices, including visual UI shape and layout generated by multiple microservices</span></span>

<span data-ttu-id="398ad-105">微服务体系结构通常从服务器端处理数据和逻辑开始。</span><span class="sxs-lookup"><span data-stu-id="398ad-105">Microservices architecture often starts with the server side handling data and logic.</span></span> <span data-ttu-id="398ad-106">但是，根据微服务设计应用程序 UI 是更好的方法。</span><span class="sxs-lookup"><span data-stu-id="398ad-106">However, a more advanced approach is to design your application UI based on microservices as well.</span></span> <span data-ttu-id="398ad-107">这意味着拥有的是由微服务生成的复合 UI，而不是服务器上的微服务和使用微服务的整体式客户端应用。</span><span class="sxs-lookup"><span data-stu-id="398ad-107">That means having a composite UI produced by the microservices, instead of having microservices on the server and just a monolithic client app consuming the microservices.</span></span> <span data-ttu-id="398ad-108">按照这种方法构建的微服务具有逻辑和可视化表示形式，功能十分完整。</span><span class="sxs-lookup"><span data-stu-id="398ad-108">With this approach, the microservices you build can be complete with both logic and visual representation.</span></span>

<span data-ttu-id="398ad-109">图 4-20 显示了在整体式客户端应用程序中使用微服务的更为简单的方式。</span><span class="sxs-lookup"><span data-stu-id="398ad-109">Figure 4-20 shows the simpler approach of just consuming microservices from a monolithic client application.</span></span> <span data-ttu-id="398ad-110">当然，在生成 HTML 之后，可以创建一个 ASP.NET MVC 服务，再生成 JavaScript。</span><span class="sxs-lookup"><span data-stu-id="398ad-110">Of course, you could have an ASP.NET MVC service in between producing the HTML and JavaScript.</span></span> <span data-ttu-id="398ad-111">该图是一张简图，强调拥有使用微服务的单一（整体式）客户端 UI，该 UI 仅关注逻辑和数据，而不关注 UI 形状（HTML 和 JavaScript）。</span><span class="sxs-lookup"><span data-stu-id="398ad-111">The figure is a simplification that highlights that you have a single (monolithic) client UI consuming the microservices, which just focus on logic and data and not on the UI shape (HTML and JavaScript).</span></span>

<span data-ttu-id="398ad-114">与之相比，复合 UI 是由微服务本身精确生成和组合成的。</span><span class="sxs-lookup"><span data-stu-id="398ad-114">In contrast, a composite UI is precisely generated and composed by the microservices themselves.</span></span> <span data-ttu-id="398ad-115">一些微服务可控制 UI 特定区域的视觉形状。</span><span class="sxs-lookup"><span data-stu-id="398ad-115">Some of the microservices drive the visual shape of specific areas of the UI.</span></span> <span data-ttu-id="398ad-116">主要区别在于，这具有基于模板的客户端 UI 组件（例如 TS 类），而这些模板的 data-shaping-UI ViewModel 来自于每个微服务。</span><span class="sxs-lookup"><span data-stu-id="398ad-116">The key difference is that you have client UI components (TS classes, for example) based on templates, and the data-shaping-UI ViewModel for those templates comes from each microservice.</span></span>

<span data-ttu-id="398ad-117">客户端应用程序启动时，每个客户端 UI 组件（例如 TypeScript 类）自身都会注册一个基础结构微服务，该微服务能够为特定方案提供 ViewModel。</span><span class="sxs-lookup"><span data-stu-id="398ad-117">At client application start-up time, each of the client UI components (TypeScript classes, for example) registers itself with an infrastructure microservice capable of providing ViewModels for a given scenario.</span></span> <span data-ttu-id="398ad-118">如果微服务形状发生更改，则 UI 也会更改。</span><span class="sxs-lookup"><span data-stu-id="398ad-118">If the microservice changes the shape, the UI changes also.</span></span>

<span data-ttu-id="398ad-119">图 4-21 显示了一种复合 UI 方法。</span><span class="sxs-lookup"><span data-stu-id="398ad-119">Figure 4-21 shows a version of this composite UI approach.</span></span> <span data-ttu-id="398ad-120">这只是一张简图，因为你可能有其他基于不同技术聚合精细部件的微服务，具体取决于是构建传统 Web 方法 (ASP.NET MVC) 还是 SPA（单页应用程序）。</span><span class="sxs-lookup"><span data-stu-id="398ad-120">This is simplified, because you might have other microservices that are aggregating granular parts based on different techniques—it depends on whether you are building a traditional web approach (ASP.NET MVC) or an SPA (Single Page Application).</span></span>

<span data-ttu-id="398ad-123">这些 UI 组合微服务中的每一个都类似于一个小型的 API 网关。</span><span class="sxs-lookup"><span data-stu-id="398ad-123">Each of those UI composition microservices would be similar to a small API Gateway.</span></span> <span data-ttu-id="398ad-124">但在此处，每个服务都负责一个小的 UI 领域。</span><span class="sxs-lookup"><span data-stu-id="398ad-124">But in this case each is responsible for a small UI area.</span></span>

<span data-ttu-id="398ad-125">由微服务驱动的复合 UI 方法的挑战性可能更大也可能更小，具体取决于使用的 UI 技术。</span><span class="sxs-lookup"><span data-stu-id="398ad-125">A composite UI approach that is driven by microservices can be more challenging or less so, depending on what UI technologies you are using.</span></span> <span data-ttu-id="398ad-126">例如，对于用于生成 SPA 或本机移动应用的传统 Web 应用程序，则不会使用相同的构建技术（如开发 Xamarin 应用时，此方法可能更具挑战性）。</span><span class="sxs-lookup"><span data-stu-id="398ad-126">For instance, you will not use the same techniques for building a traditional web application that you use for building an SPA or for native mobile app (as when developing Xamarin apps, which can be more challenging for this approach).</span></span>

<span data-ttu-id="398ad-127">出于多种原因，[eShopOnContainers](http://aka.ms/MicroservicesArchitecture) 示例应用程序使用整体式 UI 方法。</span><span class="sxs-lookup"><span data-stu-id="398ad-127">The [eShopOnContainers](http://aka.ms/MicroservicesArchitecture) sample application uses the monolithic UI approach for multiple reasons.</span></span> <span data-ttu-id="398ad-128">首先，其作用是引入微服务和容器。</span><span class="sxs-lookup"><span data-stu-id="398ad-128">First, it is an introduction to microservices and containers.</span></span> <span data-ttu-id="398ad-129">虽然复合 UI 更高级，但该 UI 的设计和开发工作也更复杂。</span><span class="sxs-lookup"><span data-stu-id="398ad-129">A composite UI is more advanced but also requires further complexity when designing and developing the UI.</span></span> <span data-ttu-id="398ad-130">其次，eShopOnContainers 还提供了基于 Xamarin 的本机移动应用，这增加了客户端 C\# 的复杂性。</span><span class="sxs-lookup"><span data-stu-id="398ad-130">Second, eShopOnContainers also provides a native mobile app based on Xamarin, which would make it more complex on the client C\# side.</span></span>
