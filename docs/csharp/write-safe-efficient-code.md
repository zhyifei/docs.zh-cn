## <a name="conclusions"></a><span data-ttu-id="e5e86-258">结论</span><span class="sxs-lookup"><span data-stu-id="e5e86-258">Conclusions</span></span>

<span data-ttu-id="e5e86-259">使用值类型可最大限度地减少分配操作的数量：</span><span class="sxs-lookup"><span data-stu-id="e5e86-259">Using value types minimizes the number of allocation operations:</span></span>

- <span data-ttu-id="e5e86-260">值类型的存储是为局部变量和方法参数分配的堆栈。</span><span class="sxs-lookup"><span data-stu-id="e5e86-260">Storage for value types is stack allocated for local variables and method arguments.</span></span>
- <span data-ttu-id="e5e86-261">作为其他对象成员的值类型的存储被分配为该对象的一部分，而不是作为单独的分配。</span><span class="sxs-lookup"><span data-stu-id="e5e86-261">Storage for value types that are members of other objects is allocated as part of that object, not as a separate allocation.</span></span>
- <span data-ttu-id="e5e86-262">值类型返回值的存储为堆栈分配。</span><span class="sxs-lookup"><span data-stu-id="e5e86-262">Storage for value type return values is stack allocated.</span></span>

<span data-ttu-id="e5e86-263">将其与相同情况下的引用类型进行对比：</span><span class="sxs-lookup"><span data-stu-id="e5e86-263">Contrast that with reference types in those same situations:</span></span>

- <span data-ttu-id="e5e86-264">引用类型的存储是为本地变量和方法参数分配的堆。</span><span class="sxs-lookup"><span data-stu-id="e5e86-264">Storage for reference types are heap allocated for local variables and method arguments.</span></span> <span data-ttu-id="e5e86-265">引用存储在堆栈中。</span><span class="sxs-lookup"><span data-stu-id="e5e86-265">The reference is stored on the stack.</span></span>
- <span data-ttu-id="e5e86-266">作为其他对象成员的引用类型的存储在堆上分别进行分配。</span><span class="sxs-lookup"><span data-stu-id="e5e86-266">Storage for reference types that are members of other objects are separately allocated on the heap.</span></span> <span data-ttu-id="e5e86-267">包含的对象存储引用。</span><span class="sxs-lookup"><span data-stu-id="e5e86-267">The containing object stores the reference.</span></span>
- <span data-ttu-id="e5e86-268">引用类型返回值的存储是堆分配的。</span><span class="sxs-lookup"><span data-stu-id="e5e86-268">Storage for reference type return values is heap allocated.</span></span> <span data-ttu-id="e5e86-269">对该存储的引用存储在堆栈中。</span><span class="sxs-lookup"><span data-stu-id="e5e86-269">The reference to that storage is stored on the stack.</span></span>

<span data-ttu-id="e5e86-270">最小化分配附带权衡。</span><span class="sxs-lookup"><span data-stu-id="e5e86-270">Minimizing allocations comes with tradeoffs.</span></span> <span data-ttu-id="e5e86-271">当 `struct` 的大小大于引用大小时，可以复制更多内存。</span><span class="sxs-lookup"><span data-stu-id="e5e86-271">You copy more memory when the size of the `struct` is larger than the size of a reference.</span></span> <span data-ttu-id="e5e86-272">引用通常为 64 位或 32 位，并且取决于目标机器 CPU。</span><span class="sxs-lookup"><span data-stu-id="e5e86-272">A reference is typically 64 bits or 32 bits, and depends on the target machine CPU.</span></span>

<span data-ttu-id="e5e86-273">这些权衡通常对性能影响最小。</span><span class="sxs-lookup"><span data-stu-id="e5e86-273">These tradeoffs generally have minimal performance impact.</span></span> <span data-ttu-id="e5e86-274">但是，对于大型结构或大型集合，性能影响会增加。</span><span class="sxs-lookup"><span data-stu-id="e5e86-274">However, for large structs or larger collections, the performance impact increases.</span></span> <span data-ttu-id="e5e86-275">对于程序的紧密循环和热路径，影响可能很大。</span><span class="sxs-lookup"><span data-stu-id="e5e86-275">The impact can be large in tight loops and hot paths for programs.</span></span>
