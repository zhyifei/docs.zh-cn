# <a name="nullable-reference-types"></a><span data-ttu-id="a6b6a-104">可为空引用类型</span><span class="sxs-lookup"><span data-stu-id="a6b6a-104">Nullable reference types</span></span>

<span data-ttu-id="a6b6a-105">C#8.0 引入了“可为空引用类型”和“不可为空引用类型”，使你能够对引用类型变量的属性作出重要声明   ：</span><span class="sxs-lookup"><span data-stu-id="a6b6a-105">C# 8.0 introduces **nullable reference types** and **non-nullable reference types** that enable you to make important statements about the properties for reference type variables:</span></span>

- <span data-ttu-id="a6b6a-106">引用不应为 null  。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-106">**A reference isn't supposed to be null**.</span></span> <span data-ttu-id="a6b6a-107">当变量不应为 null 时，编译器会强制执行规则，以确保在不首先检查它们是否为 null 的情况下，取消引用这些变量是安全的：</span><span class="sxs-lookup"><span data-stu-id="a6b6a-107">When variables aren't supposed to be null, the compiler enforces rules that ensure it's safe to dereference these variables without first checking that it isn't null:</span></span>
  - <span data-ttu-id="a6b6a-108">必须将变量初始化为非 null 值。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-108">The variable must be initialized to a non-null value.</span></span>
  - <span data-ttu-id="a6b6a-109">变量永远不能赋值为 `null`。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-109">The variable can never be assigned the value `null`.</span></span>
- <span data-ttu-id="a6b6a-110">引用可为 null  。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-110">**A reference may be null**.</span></span> <span data-ttu-id="a6b6a-111">当变量可以为 null 时，编译器会强制执行不同的规则以确保你已正确检查空引用：</span><span class="sxs-lookup"><span data-stu-id="a6b6a-111">When variables may be null, the compiler enforces different rules to ensure that you've correctly checked for a null reference:</span></span>
  - <span data-ttu-id="a6b6a-112">只有当编译器可以保证该值不为 null 时，才可以取消引用该变量。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-112">The variable may only be dereferenced when the compiler can guarantee that the value isn't null.</span></span>
  - <span data-ttu-id="a6b6a-113">这些变量可以使用默认的 `null` 值进行初始化，也可以在其他代码中赋值为 `null`。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-113">These variables may be initialized with the default `null` value and may be assigned the value `null` in other code.</span></span>

<span data-ttu-id="a6b6a-114">在 C# 的早期版本中，无法从变量声明中确定设计意图，与处理引用变量相比，这个新功能提供了显著的好处。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-114">This new feature provides significant benefits over the handling of reference variables in earlier versions of C# where the design intent can't be determined from the variable declaration.</span></span> <span data-ttu-id="a6b6a-115">编译器不提供针对引用类型的空引用异常的安全性：</span><span class="sxs-lookup"><span data-stu-id="a6b6a-115">The compiler didn't provide safety against null reference exceptions for reference types:</span></span>

- <span data-ttu-id="a6b6a-116">引用可为 null  。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-116">**A reference can be null**.</span></span> <span data-ttu-id="a6b6a-117">将引用类型初始化为 null 或稍后将其指定为 null 时，编译器不会发出警告。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-117">The compiler doesn't issue warnings when a reference type is initialized to null, or later assigned to null.</span></span> <span data-ttu-id="a6b6a-118">在没有进行 null 检查的情况下取消引用这些变量，编译器会发出警告。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-118">The compiler issues warnings when these variables are dereferenced without null checks.</span></span>
- <span data-ttu-id="a6b6a-119">假定引用不为 null  。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-119">**A reference is assumed to be not null**.</span></span> <span data-ttu-id="a6b6a-120">当引用类型被取消引用时，编译器不会发出任何警告。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-120">The compiler doesn't issue any warnings when reference types are dereferenced.</span></span> <span data-ttu-id="a6b6a-121">如果将变量设置为可以为 null 的表达式，则编译器会发出警告。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-121">The compiler issues warnings if a variable is set to an expression that may be null.</span></span>

<span data-ttu-id="a6b6a-122">将在编译时发出这些警告。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-122">These warnings are emitted at compile time.</span></span> <span data-ttu-id="a6b6a-123">编译器不会在可为 null 的上下文中添加任何 null 检查或其他运行时构造。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-123">The compiler doesn't add any null checks or other runtime constructs in a nullable context.</span></span> <span data-ttu-id="a6b6a-124">在运行时，可为 null 的引用和不可为 null 的引用是等效的。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-124">At runtime, a nullable reference and a non-nullable reference are equivalent.</span></span>

<span data-ttu-id="a6b6a-125">通过添加可为空引用类型，你可以更清楚地声明你的意图。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-125">With the addition of nullable reference types, you can declare your intent more clearly.</span></span> <span data-ttu-id="a6b6a-126">`null` 值是表示变量不引用值的正确方法。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-126">The `null` value is the correct way to represent that a variable doesn't refer to a value.</span></span> <span data-ttu-id="a6b6a-127">请勿使用此功能从代码中删除所有 `null` 值。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-127">Don't use this feature to remove all `null` values from your code.</span></span> <span data-ttu-id="a6b6a-128">相反，应该向编译器和其他读取代码的开发人员声明你的意图。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-128">Rather, you should declare your intent to the compiler and other developers that read your code.</span></span> <span data-ttu-id="a6b6a-129">通过声明意图，编译器会在你编写与该意图不一致的代码时通知你。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-129">By declaring your intent, the compiler informs you when you write code that is inconsistent with that intent.</span></span>

<span data-ttu-id="a6b6a-130">使用与[可为空值类型](language-reference/builtin-types/nullable-value-types.md)相同的语法记录**可为空引用类型**：将 `?` 附加到变量的类型。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-130">A **nullable reference type** is noted using the same syntax as [nullable value types](language-reference/builtin-types/nullable-value-types.md): a `?` is appended to the type of the variable.</span></span> <span data-ttu-id="a6b6a-131">例如，以下变量声明表示可为空的字符串变量 `name`：</span><span class="sxs-lookup"><span data-stu-id="a6b6a-131">For example, the following variable declaration represents a nullable string variable, `name`:</span></span>

```csharp

```

<span data-ttu-id="a6b6a-132">未将 `?` 附加到类型名称的任何变量都是“不可为 null 引用类型”  。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-132">Any variable where the `?` isn't appended to the type name is a **non-nullable reference type**.</span></span> <span data-ttu-id="a6b6a-133">这包括启用此功能时现有代码中的所有引用类型变量。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-133">That includes all reference type variables in existing code when you've enabled this feature.</span></span>

<span data-ttu-id="a6b6a-134">编译器使用静态分析来确定可为空引用是否为非 null。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-134">The compiler uses static analysis to determine if a nullable reference is known to be non-null.</span></span> <span data-ttu-id="a6b6a-135">如果你在一个可为空引用可能是 null 时对其取消引用，编译器将向你发出警告。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-135">The compiler warns you if you dereference a nullable reference when it may be null.</span></span> <span data-ttu-id="a6b6a-136">可以通过使用 [NULL 包容运算符](language-reference/operators/null-forgiving.md) `!` 后跟变量名称来替代此行为。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-136">You can override this behavior by using the [null-forgiving operator](language-reference/operators/null-forgiving.md) `!` following a variable name.</span></span> <span data-ttu-id="a6b6a-137">例如，若知道 `name` 变量不为 null 但编译器仍发出警告，则可以编写以下代码来覆盖编译器的分析：</span><span class="sxs-lookup"><span data-stu-id="a6b6a-137">For example, if you know the `name` variable isn't null but the compiler issues a warning, you can write the following code to override the compiler's analysis:</span></span>


## <a name="nullability-of-types"></a><span data-ttu-id="a6b6a-138">类型为 Null 性</span><span class="sxs-lookup"><span data-stu-id="a6b6a-138">Nullability of types</span></span>

<span data-ttu-id="a6b6a-139">任何引用类型都可以具有四个“为 Null 性”中的一个，它描述了何时生成警告  ：</span><span class="sxs-lookup"><span data-stu-id="a6b6a-139">Any reference type can have one of four *nullabilities*, which describes when warnings are generated:</span></span>

- <span data-ttu-id="a6b6a-140">*不可为空*：无法将 null 分配给此类型的变量。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-140">*Nonnullable*: Null can't be assigned to variables of this type.</span></span> <span data-ttu-id="a6b6a-141">在取消引用之前，无需对此类型的变量进行 null 检查。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-141">Variables of this type don't need to be null-checked before dereferencing.</span></span>
- <span data-ttu-id="a6b6a-142">*可为空*：可将 null 分配给此类型的变量。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-142">*Nullable*: Null can be assigned to variables of this type.</span></span> <span data-ttu-id="a6b6a-143">在不首先检查 `null` 的情况下取消引用此类型的变量时发出警告。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-143">Dereferencing variables of this type without first checking for `null` causes a warning.</span></span>
- <span data-ttu-id="a6b6a-144">*无视*：“无视”是 C# 8.0 之前版本的状态。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-144">*Oblivious*: Oblivious is the pre-C# 8.0 state.</span></span> <span data-ttu-id="a6b6a-145">可以取消引用或分配此类型的变量而不发出警告。</span><span class="sxs-lookup"><span data-stu-id="a6b6a-145">Variables of this type can be dereferenced or assigned without warnings.</span></span>
