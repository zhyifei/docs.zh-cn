# <a name="language-integrated-query-linq"></a><span data-ttu-id="ea1dd-102">语言集成查询 (LINQ)</span><span class="sxs-lookup"><span data-stu-id="ea1dd-102">Language Integrated Query (LINQ)</span></span>

<span data-ttu-id="ea1dd-103">语言集成查询 (LINQ) 是一系列直接将查询功能集成到 C# 语言的技术统称。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-103">Language-Integrated Query (LINQ) is the name for a set of technologies based on the integration of query capabilities directly into the C# language.</span></span> <span data-ttu-id="ea1dd-104">数据查询历来都表示为简单的字符串，没有编译时类型检查或 IntelliSense 支持。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-104">Traditionally, queries against data are expressed as simple strings without type checking at compile time or IntelliSense support.</span></span> <span data-ttu-id="ea1dd-105">此外，需要针对每种类型的数据源了解不同的查询语言：SQL 数据库、XML 文档、各种 Web 服务等。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-105">Furthermore, you have to learn a different query language for each type of data source: SQL databases, XML documents, various Web services, and so on.</span></span> <span data-ttu-id="ea1dd-106">借助 LINQ，查询成为了最高级的语言构造，就像类、方法和事件一样。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-106">With LINQ, a query is a first-class language construct, just like classes, methods, events.</span></span> <span data-ttu-id="ea1dd-107">可以使用语言关键字和熟悉的运算符针对强类型化对象集合编写查询。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-107">You write queries against strongly typed collections of objects by using language keywords and familiar operators.</span></span>  <span data-ttu-id="ea1dd-108">LINQ 系列技术提供了针对对象 (LINQ to Objects)、关系数据库 (LINQ to SQL) 和 XML (LINQ to XML) 的一致查询体验。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-108">The LINQ family of technologies provides a consistent query experience for objects (LINQ to Objects), relational databases (LINQ to SQL), and XML (LINQ to XML).</span></span>  

<span data-ttu-id="ea1dd-109">对于编写查询的开发者来说，LINQ 最明显的“语言集成”部分就是查询表达式。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-109">For a developer who writes queries, the most visible "language-integrated" part of LINQ is the query expression.</span></span> <span data-ttu-id="ea1dd-110">查询表达式采用声明性*查询语法*编写而成。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-110">Query expressions are written in a declarative *query syntax*.</span></span> <span data-ttu-id="ea1dd-111">使用查询语法，可以用最少的代码对数据源执行筛选、排序和分组操作。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-111">By using query syntax, you can perform filtering, ordering, and grouping operations on data sources with a minimum of code.</span></span> <span data-ttu-id="ea1dd-112">可使用相同的基本查询表达式模式来查询和转换 SQL 数据库、ADO .NET 数据集、XML 文档和流以及 .NET 集合中的数据。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-112">You use the same basic query expression patterns to query and transform data in SQL databases, ADO .NET Datasets, XML documents and streams, and .NET collections.</span></span>

<span data-ttu-id="ea1dd-113">在 C# 中可为以下对象编写 LINQ 查询：SQL Server 数据库、XML 文档、ADO.NET 数据集以及支持 <xref:System.Collections.IEnumerable> 或泛型 <xref:System.Collections.Generic.IEnumerable%601> 接口的任何对象集合。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-113">You can write LINQ queries in C# for SQL Server databases, XML documents, ADO.NET Datasets, and any collection of objects that supports <xref:System.Collections.IEnumerable> or the generic <xref:System.Collections.Generic.IEnumerable%601> interface.</span></span> <span data-ttu-id="ea1dd-114">此外，第三方也为许多 Web 服务和其他数据库实现提供了 LINQ 支持。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-114">LINQ support is also provided by third parties for many Web services and other database implementations.</span></span>

## <a name="query-expression-overview"></a><span data-ttu-id="ea1dd-119">查询表达式概述</span><span class="sxs-lookup"><span data-stu-id="ea1dd-119">Query expression overview</span></span>

- <span data-ttu-id="ea1dd-120">查询表达式可用于查询并转换所有启用了 LINQ 的数据源中的数据。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-120">Query expressions can be used to query and to transform data from any LINQ-enabled data source.</span></span> <span data-ttu-id="ea1dd-121">例如，通过一个查询即可检索 SQL 数据库中的数据，并生成 XML 流作为输出。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-121">For example, a single query can retrieve data from a SQL database, and produce an XML stream as output.</span></span>  
  
- <span data-ttu-id="ea1dd-122">查询表达式易于掌握，因为使用了许多熟悉的 C# 语言构造。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-122">Query expressions are easy to master because they use many familiar C# language constructs.</span></span>  
  
- <span data-ttu-id="ea1dd-123">查询表达式中的变量全都是强类型，尽管在许多情况下，无需显式提供类型，因为编译器可以推断出。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-123">The variables in a query expression are all strongly typed, although in many cases you do not have to provide the type explicitly because the compiler can infer it.</span></span> <span data-ttu-id="ea1dd-124">有关详细信息，请参阅 [LINQ 查询操作中的类型关系](type-relationships-in-linq-query-operations.md)。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-124">For more information, see [Type relationships in LINQ query operations](type-relationships-in-linq-query-operations.md).</span></span>  
  
- <span data-ttu-id="ea1dd-125">只有在循环访问查询变量后，才会执行查询（例如，在 `foreach` 语句中）。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-125">A query is not executed until you iterate over the query variable, for example, in a `foreach` statement.</span></span> <span data-ttu-id="ea1dd-126">有关详细信息，请参阅 [LINQ 查询简介](introduction-to-linq-queries.md)。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-126">For more information, see [Introduction to LINQ queries](introduction-to-linq-queries.md).</span></span>  
  
- <span data-ttu-id="ea1dd-127">在编译时，查询表达式根据 C# 规范规则转换成标准查询运算符方法调用。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-127">At compile time, query expressions are converted to Standard Query Operator method calls according to the rules set forth in the C# specification.</span></span> <span data-ttu-id="ea1dd-128">可使用查询语法表示的任何查询都可以使用方法语法进行表示。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-128">Any query that can be expressed by using query syntax can also be expressed by using method syntax.</span></span> <span data-ttu-id="ea1dd-129">不过，在大多数情况下，查询语法的可读性更高，也更为简洁。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-129">However, in most cases query syntax is more readable and concise.</span></span> <span data-ttu-id="ea1dd-130">有关详细信息，请参阅 [C# 语言规范](~/_csharplang/spec/expressions.md#query-expressions)和[标准查询运算符概述](standard-query-operators-overview.md)。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-130">For more information, see [C# language specification](~/_csharplang/spec/expressions.md#query-expressions) and [Standard query operators overview](standard-query-operators-overview.md).</span></span>  
  
- <span data-ttu-id="ea1dd-131">通常，我们建议在编写 LINQ 查询时尽量使用查询语法，并在必要时尽可能使用方法语法。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-131">As a rule when you write LINQ queries, we recommend that you use query syntax whenever possible and method syntax whenever necessary.</span></span> <span data-ttu-id="ea1dd-132">这两种不同的形式在语义或性能上毫无差异。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-132">There is no semantic or performance difference between the two different forms.</span></span> <span data-ttu-id="ea1dd-133">查询表达式通常比使用方法语法编写的等同表达式更具可读性。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-133">Query expressions are often more readable than equivalent expressions written in method syntax.</span></span>  
  
- <span data-ttu-id="ea1dd-134">一些查询操作（如 <xref:System.Linq.Enumerable.Count%2A> 或 <xref:System.Linq.Enumerable.Max%2A>）没有等效的查询表达式子句，因此必须表示为方法调用。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-134">Some query operations, such as <xref:System.Linq.Enumerable.Count%2A> or <xref:System.Linq.Enumerable.Max%2A>, have no equivalent query expression clause and must therefore be expressed as a method call.</span></span> <span data-ttu-id="ea1dd-135">可以各种方式结合使用方法语法和查询语法。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-135">Method syntax can be combined with query syntax in various ways.</span></span> <span data-ttu-id="ea1dd-136">有关详细信息，请参阅 [LINQ 中的查询语法和方法语法](query-syntax-and-method-syntax-in-linq.md)。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-136">For more information, see [Query syntax and method syntax in LINQ](query-syntax-and-method-syntax-in-linq.md).</span></span>  
  
- <span data-ttu-id="ea1dd-137">查询表达式可被编译成表达式树或委托，具体视应用查询的类型而定。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-137">Query expressions can be compiled to expression trees or to delegates, depending on the type that the query is applied to.</span></span> <span data-ttu-id="ea1dd-138"><xref:System.Collections.Generic.IEnumerable%601> 查询编译为委托。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-138"><xref:System.Collections.Generic.IEnumerable%601> queries are compiled to delegates.</span></span> <span data-ttu-id="ea1dd-139"><xref:System.Linq.IQueryable> 和 <xref:System.Linq.IQueryable%601> 查询编译为表达式树。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-139"><xref:System.Linq.IQueryable> and <xref:System.Linq.IQueryable%601> queries are compiled to expression trees.</span></span> <span data-ttu-id="ea1dd-140">有关详细信息，请参阅[表达式树](../../../expression-trees.md)。</span><span class="sxs-lookup"><span data-stu-id="ea1dd-140">For more information, see [Expression trees](../../../expression-trees.md).</span></span>
