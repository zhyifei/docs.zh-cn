## <a name="net-compiler-platform-sdk-concepts"></a><span data-ttu-id="48a49-112">.NET Compiler Platform SDK 概念</span><span class="sxs-lookup"><span data-stu-id="48a49-112">.NET Compiler Platform SDK concepts</span></span>

<span data-ttu-id="48a49-115">使用 .NET Compiler Platform SDK，可以生成分析器和代码修补程序，从而发现和更正编码错误。</span><span class="sxs-lookup"><span data-stu-id="48a49-115">The .NET Compiler Platform SDK enables you to build ***analyzers*** and ***code fixes*** that find and correct coding mistakes.</span></span> <span data-ttu-id="48a49-116">分析器不仅理解代码的语法和结构，还能检测应更正的做法。</span><span class="sxs-lookup"><span data-stu-id="48a49-116">***Analyzers*** understand the syntax and structure of code and detect practices that should be corrected.</span></span> <span data-ttu-id="48a49-117">代码修补程序建议一处或多处修复，以修复分析器发现的编码错误。</span><span class="sxs-lookup"><span data-stu-id="48a49-117">***Code fixes*** provide one or more suggested fixes for addressing coding mistakes found by analyzers.</span></span> <span data-ttu-id="48a49-118">通常情况下，分析器和关联的代码修补程序一起打包在一个项目中。</span><span class="sxs-lookup"><span data-stu-id="48a49-118">Typically, an analyzer and the associated code fixes are packaged together in a single project.</span></span> 

<span data-ttu-id="48a49-119">分析器和代码修补程序通过静态分析来理解代码。</span><span class="sxs-lookup"><span data-stu-id="48a49-119">Analyzers and code fixes use static analysis to understand code.</span></span> <span data-ttu-id="48a49-120">它们既不运行代码，也未带来其他测试方面的好处。</span><span class="sxs-lookup"><span data-stu-id="48a49-120">They do not run the code or provide other testing benefits.</span></span> <span data-ttu-id="48a49-121">但是，它们可以指出经常导致 bug 的做法、无法维护的代码或标准指导性验证。</span><span class="sxs-lookup"><span data-stu-id="48a49-121">They can, however, point out practices that often lead to bugs, unmaintainable code, or standard guideline validation.</span></span>

<span data-ttu-id="48a49-122">.NET Compiler Platform SDK 提供了一组 API，可便于检查和理解 C# 或 Visual Basic 基准代码。</span><span class="sxs-lookup"><span data-stu-id="48a49-122">The .NET Compiler Platform SDK provides a single set of APIs that enable you to examine and understand a C# or Visual Basic codebase.</span></span> <span data-ttu-id="48a49-123">由于可以使用这一基准代码，因此能够利用 .NET Compiler Platform SDK 提供的语法和语义分析 API，更轻松地编写分析器和代码修补程序。</span><span class="sxs-lookup"><span data-stu-id="48a49-123">Because you can use this single codebase, you can write analyzers and code fixes more easily by leveraging the syntactic and semantic analysis APIs provided by the .NET Compiler Platform SDK.</span></span> <span data-ttu-id="48a49-124">从复制由编译器执行的分析的繁重任务中解放出来，可以将精力放在为项目或库查找常见编码错误并进行修复的更为重要的任务上。</span><span class="sxs-lookup"><span data-stu-id="48a49-124">Freed from the large task of replicating the analysis done by the compiler, you can concentrate on the more focused task of finding and fixing common coding errors for your project or library.</span></span>

<span data-ttu-id="48a49-125">这带来的一个小小的好处是，分析器和代码修补程序较小，在 Visual Studio 中加载时占用的内存比为了理解项目中的代码而编写自己的基准代码占用的内存少得多。</span><span class="sxs-lookup"><span data-stu-id="48a49-125">A smaller benefit is that your analyzers and code fixes are smaller and use much less memory when loaded in Visual Studio than they would if you wrote your own codebase to understand the code in a project.</span></span> <span data-ttu-id="48a49-126">利用编译器和 Visual Studio 使用的相同类，可以创建自己的静态分析工具。</span><span class="sxs-lookup"><span data-stu-id="48a49-126">By leveraging the same classes used by the compiler and Visual Studio, you can create your own static analysis tools.</span></span> <span data-ttu-id="48a49-127">也就是说，团队可以使用分析器和代码修补程序，而不会对 IDE 的性能造成显著影响。</span><span class="sxs-lookup"><span data-stu-id="48a49-127">This means your team can use analyzers and code fixes without a noticeable impact on the IDE's performance.</span></span>
