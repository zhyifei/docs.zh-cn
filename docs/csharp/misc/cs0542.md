# <a name="compiler-error-cs0542"></a><span data-ttu-id="adeab-102">编译器错误 CS0542</span><span class="sxs-lookup"><span data-stu-id="adeab-102">Compiler Error CS0542</span></span>
<span data-ttu-id="adeab-103">“用户定义类型”: 成员名称不能与它们的封闭类型相同</span><span class="sxs-lookup"><span data-stu-id="adeab-103">'user-defined type' : member names cannot be the same as their enclosing type</span></span>  
  
 <span data-ttu-id="adeab-104">类或结构的成员不能与类或结构同名，除非该成员是一个构造函数。</span><span class="sxs-lookup"><span data-stu-id="adeab-104">The members of a class or struct cannot have the same name as the class or struct, unless the member is a constructor.</span></span>  
  
 <span data-ttu-id="adeab-105">下面的示例生成 CS0542:</span><span class="sxs-lookup"><span data-stu-id="adeab-105">The following sample generates CS0542:</span></span>  
  
```csharp  
// CS0542.cs  
class C  
{  
    public int C;  
}  
```  
  
 <span data-ttu-id="adeab-106">如果无意中将返回类型放在构造函数中（实际上会使其成为普通方法），则可能导致此错误。</span><span class="sxs-lookup"><span data-stu-id="adeab-106">This error might be caused if you inadvertently put a return type on a constructor, which in effect makes it into an ordinary method.</span></span> <span data-ttu-id="adeab-107">下面的示例将生成 CS0542，因为 `F` 是一种方法，不是构造函数，因为它具有返回类型：</span><span class="sxs-lookup"><span data-stu-id="adeab-107">The following example generates CS0542 because `F` is a method, not a constructor, because it has a return type:</span></span>  
  
```csharp  
// CS0542.cs  
class F  
{  
   // Remove void from F() to resolve the problem.  
   void F()   // CS0542, same name as the class  
   {  
   }  
}  
  
class MyClass  
{  
   public static void Main()  
   {  
   }  
}  
```  
  
 <span data-ttu-id="adeab-108">如果类命名为“Item”，并且具有一个声明为 `this`的索引器 ，则可能出现此错误。</span><span class="sxs-lookup"><span data-stu-id="adeab-108">If your class is named 'Item' and has an indexer declared as `this`, you may get this error.</span></span> <span data-ttu-id="adeab-109">默认索引器在发出的代码中被命名为“Item”，从而导致冲突。</span><span class="sxs-lookup"><span data-stu-id="adeab-109">A default indexer is given the name 'Item' in the emitted code, creating the conflict.</span></span>  
  
```csharp  
// CS0542b.cs  
class Item  
{  
   public int this[int i]  // CS0542  
   {  
      get  
      {  
         return 0;  
      }  
   }  
}  
  
class CMain  
{  
   public static void Main()  
   {  
   }  
}  
```
