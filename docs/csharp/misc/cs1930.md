# <a name="compiler-error-cs1930"></a><span data-ttu-id="6aa9c-102">编译器错误 CS1930</span><span class="sxs-lookup"><span data-stu-id="6aa9c-102">Compiler Error CS1930</span></span>
<span data-ttu-id="6aa9c-103">已声明范围变量“name”</span><span class="sxs-lookup"><span data-stu-id="6aa9c-103">The range variable 'name' has already been declared</span></span>  
  
 <span data-ttu-id="6aa9c-104">查询表达式中的范围变量会一直位于范围内，直到该查询表达式终止。</span><span class="sxs-lookup"><span data-stu-id="6aa9c-104">The range variable in a query expression is in scope until the query expression terminates.</span></span> <span data-ttu-id="6aa9c-105">因此，该变量必须具有唯一标识符。</span><span class="sxs-lookup"><span data-stu-id="6aa9c-105">It must therefore have a unique identifier.</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="6aa9c-106">更正此错误</span><span class="sxs-lookup"><span data-stu-id="6aa9c-106">To correct this error</span></span>  
  
1.  <span data-ttu-id="6aa9c-107">为在查询表达式中引入的每个范围变量指定一个唯一名称。</span><span class="sxs-lookup"><span data-stu-id="6aa9c-107">Give a unique name to each range variable that is introduced in a query expression.</span></span>  
  
## <a name="example"></a><span data-ttu-id="6aa9c-108">示例</span><span class="sxs-lookup"><span data-stu-id="6aa9c-108">Example</span></span>  
 <span data-ttu-id="6aa9c-109">下面的示例生成 CS1930，因为同时对 `num` 子句中的范围变量和 `from` 子句所引入的范围变量使用了标识符 `let` 。</span><span class="sxs-lookup"><span data-stu-id="6aa9c-109">The following example generates CS1930 because the identifier `num` is used for the range variable in the `from` clause and for the range variable introduced by the `let` clause.</span></span>  
  
```csharp  
// cs1930.cs  
using System.Linq;  
class Program  
{  
    static void Main()  
    {  
        int[] nums = { 0, 1, 2, 3, 4, 5 };  
        var query = from num in nums  
                    let num = 3 // CS1930  
                    select num;   
    }  
}  
```
