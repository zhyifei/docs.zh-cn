# <a name="compiler-error-cs1104"></a><span data-ttu-id="e0f2a-102">编译器错误 CS1104</span><span class="sxs-lookup"><span data-stu-id="e0f2a-102">Compiler Error CS1104</span></span>
<span data-ttu-id="e0f2a-103">参数数组不能与“this”修饰符一起在扩展方法中使用。</span><span class="sxs-lookup"><span data-stu-id="e0f2a-103">A parameter array cannot be used with 'this' modifier on an extension method.</span></span>  
  
 <span data-ttu-id="e0f2a-104">扩展方法的第一个参数不能是参数数组。</span><span class="sxs-lookup"><span data-stu-id="e0f2a-104">The first parameter of an extension method cannot be a params array.</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="e0f2a-105">更正此错误</span><span class="sxs-lookup"><span data-stu-id="e0f2a-105">To correct this error</span></span>  
  
1.  <span data-ttu-id="e0f2a-106">请记住，扩展方法的第一个参数指定方法要扩展哪个类型。</span><span class="sxs-lookup"><span data-stu-id="e0f2a-106">Remember that the first parameter of an extension method definition specifies which type the method will "extend".</span></span> <span data-ttu-id="e0f2a-107">不是输入参数。</span><span class="sxs-lookup"><span data-stu-id="e0f2a-107">It is not an input parameter.</span></span> <span data-ttu-id="e0f2a-108">因此，此位置的参数数组没有意义。</span><span class="sxs-lookup"><span data-stu-id="e0f2a-108">Therefore, it makes no sense to have a params array in this location.</span></span> <span data-ttu-id="e0f2a-109">如果确实必须传递参数数组，应作为第二个参数传递。</span><span class="sxs-lookup"><span data-stu-id="e0f2a-109">If you do have to pass in a params array, make it the second parameter.</span></span>  
  
## <a name="example"></a><span data-ttu-id="e0f2a-110">示例</span><span class="sxs-lookup"><span data-stu-id="e0f2a-110">Example</span></span>
