## <a name="operator-associativity"></a><span data-ttu-id="debbb-148">运算符结合性</span><span class="sxs-lookup"><span data-stu-id="debbb-148">Operator associativity</span></span>

<span data-ttu-id="debbb-149">当运算符的优先级相同，运算符的结合性决定了运算的执行顺序：</span><span class="sxs-lookup"><span data-stu-id="debbb-149">When operators have the same precedence, associativity of the operators determines the order in which the operations are performed:</span></span>

- <span data-ttu-id="debbb-150">左结合运算符按从左到右的顺序计算。 </span><span class="sxs-lookup"><span data-stu-id="debbb-150">*Left-associative* operators are evaluated in order from left to right.</span></span> <span data-ttu-id="debbb-151">除[赋值运算符](assignment-operator.md)和 [null 合并运算符](null-coalescing-operator.md)外，所有二元运算符都是左结合运算符。</span><span class="sxs-lookup"><span data-stu-id="debbb-151">Except for the [assignment operators](assignment-operator.md) and the [null-coalescing operators](null-coalescing-operator.md), all binary operators are left-associative.</span></span> <span data-ttu-id="debbb-152">例如，`a + b - c` 将计算为 `(a + b) - c`。</span><span class="sxs-lookup"><span data-stu-id="debbb-152">For example, `a + b - c` is evaluated as `(a + b) - c`.</span></span>
- <span data-ttu-id="debbb-153">右结合运算符按从右到左的顺序计算。 </span><span class="sxs-lookup"><span data-stu-id="debbb-153">*Right-associative* operators are evaluated in order from right to left.</span></span> <span data-ttu-id="debbb-154">赋值运算符、null 合并运算符和[条件运算符`?:`](conditional-operator.md)是右结合运算符。</span><span class="sxs-lookup"><span data-stu-id="debbb-154">The assignment operators, the null-coalescing operators, and the [conditional operator `?:`](conditional-operator.md) are right-associative.</span></span> <span data-ttu-id="debbb-155">例如，`x = y = z` 将计算为 `x = (y = z)`。</span><span class="sxs-lookup"><span data-stu-id="debbb-155">For example, `x = y = z` is evaluated as `x = (y = z)`.</span></span>
