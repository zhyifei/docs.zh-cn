# <a name="compiler-warning-level-3-cs0675"></a><span data-ttu-id="be263-102">编译器警告（等级 3）CS0675</span><span class="sxs-lookup"><span data-stu-id="be263-102">Compiler Warning (level 3) CS0675</span></span>
<span data-ttu-id="be263-103">在经符号扩展的操作数上使用了按位“或”运算符；请考虑首先强制转换为较小的无符号类型</span><span class="sxs-lookup"><span data-stu-id="be263-103">Bitwise-or operator used on a sign-extended operand; consider casting to a smaller unsigned type first</span></span>  
  
 <span data-ttu-id="be263-104">编译器对某个变量进行了隐式拓展和带符号扩展，然后在按位或运算中使用生成的值。</span><span class="sxs-lookup"><span data-stu-id="be263-104">The compiler implicitly widened and sign-extended a variable, and then used the resulting value in a bitwise OR operation.</span></span> <span data-ttu-id="be263-105">这可能会导致意外行为。</span><span class="sxs-lookup"><span data-stu-id="be263-105">This can result in unexpected behavior.</span></span>  
  
 <span data-ttu-id="be263-106">下面的示例生成 CS0675：</span><span class="sxs-lookup"><span data-stu-id="be263-106">The following sample generates CS0675:</span></span>  
  
```csharp  
// CS0675.cs  
// compile with: /W:3  
using System;  
  
public class sign  
{  
   public static void Main()  
   {  
      int hi = 1;  
      int lo = 1;  
      long value = (((long)hi) << 32) | lo;              // CS0675  
      // try the following line instead  
      // long value = (((long)hi) << 32) | ((uint)lo);   // correct  
   }  
}  
```
