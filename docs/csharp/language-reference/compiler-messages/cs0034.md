# <a name="compiler-error-cs0034"></a><span data-ttu-id="49fab-102">编译器错误 CS0034</span><span class="sxs-lookup"><span data-stu-id="49fab-102">Compiler Error CS0034</span></span>

<span data-ttu-id="49fab-103">运算符“operator”对于“type1”和“type2”类型的操作数具有二义性</span><span class="sxs-lookup"><span data-stu-id="49fab-103">Operator 'operator' is ambiguous on operands of type 'type1' and 'type2'</span></span>

 <span data-ttu-id="49fab-104">运算符用在了两个对象上，并且编译器找到多个转换。</span><span class="sxs-lookup"><span data-stu-id="49fab-104">An operator was used on two objects and the compiler found more than one conversion.</span></span> <span data-ttu-id="49fab-105">由于转换必须是唯一的，因此必须进行强制转换，或者使其中一个转换成为显式转换。</span><span class="sxs-lookup"><span data-stu-id="49fab-105">Because conversions have to be unique, you either have to make a cast or to make one of the conversions explicit.</span></span> <span data-ttu-id="49fab-106">有关详细信息，请参阅[用户定义转换运算符](../operators/user-defined-conversion-operators.md)。</span><span class="sxs-lookup"><span data-stu-id="49fab-106">For more information, see [User-defined conversion operators](../operators/user-defined-conversion-operators.md).</span></span>

 <span data-ttu-id="49fab-107">下面的示例生成 CS0034：</span><span class="sxs-lookup"><span data-stu-id="49fab-107">The following sample generates CS0034:</span></span>

```csharp
// CS0034.cs
public class A
{
    // Allows for the conversion of A object to int.
    public static implicit operator int (A s)
    {
        return 0;
    }

    public static implicit operator string (A i)
    {
        return null;
    }
}

public class B
{
    public static implicit operator int (B s)  
    // One way to resolve this CS0034 is to make one conversion explicit.
    // public static explicit operator int (B s)
    {
        return 0;
    }

    public static implicit operator string (B i)
    {
        return null;
    }

    public static implicit operator B (string i)
    {
        return null;
    }

    public static implicit operator B (int i)
    {
        return null;
    }
}

public class C
{
    public static void Main()
    {
        A a = new A();
        B b = new B();
        b = b + a;   // CS0034
        // Another way to resolve this CS0034 is to make a cast.
        // b = b + (int)a;
    }
}
```

 <span data-ttu-id="49fab-108">在 C# 1.1 中，一个编译器 bug 使定义类（该类具有对 `int` 和 `bool` 的用户定义的隐式转换）和对该类型对象使用按位 `AND` 运算符 (`&`) 成为可能。</span><span class="sxs-lookup"><span data-stu-id="49fab-108">In C# 1.1, a compiler bug made it possible to define a class that has implicit user-defined conversions to both `int` and `bool`, and to use a bitwise `AND` operator (`&`) on objects of that type.</span></span> <span data-ttu-id="49fab-109">在 C# 2.0 中，已修复此 bug，从而使编译器符合 C# 规范，因此以下代码现在将导致 CS0034：</span><span class="sxs-lookup"><span data-stu-id="49fab-109">In C# 2.0, this bug was fixed to bring the compiler into compliance with the C# specification, and therefore the following code will now cause CS0034:</span></span>

```csharp
namespace CS0034
{
    class TestClass2
    {
        public void Test()
        {
            TestClass o1 = new TestClass();
            TestClass o2 = new TestClass();
            TestClass o3 = o1 & o2; //CS0034
        }
    }

    class TestClass
    {
        public static implicit operator int(TestClass o)
        {
            return 1;
        }

        public static implicit operator TestClass(int v)
        {
            return new TestClass();
        }

        public static implicit operator bool(TestClass o)
        {
            return true;
        }
    }

}
```
