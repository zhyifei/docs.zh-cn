<span data-ttu-id="c4e07-103">如果程序集引用“Assembly Name #1”与“Assembly Name #2”一致，则可能需要提供运行时策略</span><span class="sxs-lookup"><span data-stu-id="c4e07-103">Assuming assembly reference "Assembly Name #1" matches "Assembly Name #2", you may need to supply runtime policy</span></span>  
  
 <span data-ttu-id="c4e07-104">这两个程序集的发行号和/或版本号不同。</span><span class="sxs-lookup"><span data-stu-id="c4e07-104">The two assemblies differ in release and/or version number.</span></span> <span data-ttu-id="c4e07-105">要实现统一，必须在应用程序的 .config 文件中指定指令，并且必须提供程序集正确的强名称，如下面的示例代码所示。</span><span class="sxs-lookup"><span data-stu-id="c4e07-105">For unification to occur, you must specify directives in the application's .config file, and you must provide the correct strong name of an assembly, as demonstrated in the following example code.</span></span>  
  
## <a name="example"></a><span data-ttu-id="c4e07-106">示例</span><span class="sxs-lookup"><span data-stu-id="c4e07-106">Example</span></span>  
 <span data-ttu-id="c4e07-107">下面的多文件示例使用两个不同的外部别名引用一个程序集。</span><span class="sxs-lookup"><span data-stu-id="c4e07-107">The following multifile sample references an assembly using two different external aliases.</span></span> <span data-ttu-id="c4e07-108">第一个示例生成较旧版本的代码，该代码创建程序集 CS1701_d。</span><span class="sxs-lookup"><span data-stu-id="c4e07-108">This first sample builds the older version of the code that creates assembly CS1701_d.</span></span>  
  
```csharp

// compile with: /target:library /out:cs1701_d.dll /keyfile:mykey.snk  
using System.Reflection;  
[assembly:AssemblyVersion("1.0")]  
public class A {  
   public void M1() {}  
}  
  
public class C1 {}  
```  
  
## <a name="example"></a><span data-ttu-id="c4e07-109">示例</span><span class="sxs-lookup"><span data-stu-id="c4e07-109">Example</span></span>  
 <span data-ttu-id="c4e07-110">这是用于创建较新版本的程序集 CS1701_d 的代码。</span><span class="sxs-lookup"><span data-stu-id="c4e07-110">This is the code that creates the newer version of assembly CS1701_d.</span></span> <span data-ttu-id="c4e07-111">注意，由于这两个版本的输出文件同名，因此它需要将较新版本编译到不同于较早版本的目录中。</span><span class="sxs-lookup"><span data-stu-id="c4e07-111">Note that it compiles into a different directory than the older version, necessary since the output files have the same names.</span></span>

```csharp  
// compile with: /target:library /out:c:\\cs1701_d.dll /keyfile:mykey.snk  
using System.Reflection;  
[assembly:AssemblyVersion("2.0")]  
public class A {  
   public void M2() {}  
   public void M1() {}  
}  
  
public class C2 {}  
public class C1 {}  
```  
  
## <a name="example"></a><span data-ttu-id="c4e07-112">示例</span><span class="sxs-lookup"><span data-stu-id="c4e07-112">Example</span></span>  
 <span data-ttu-id="c4e07-113">此示例设置外部别名 A1 和 A2。</span><span class="sxs-lookup"><span data-stu-id="c4e07-113">This sample sets up the external aliases A1 and A2.</span></span>  
  
```csharp  
// CS1701_c.cs  
// compile with: /target:library /reference:A2=c:\\cs1701_d.dll /reference:A1=cs1701_d.dll  
  
extern alias A1;  
extern alias A2;  
// using System;  
using a1 = A1::A;  
using a2 = A2::A;  
  
public class Ref {  
   public static a1 A1() { return new a1(); }  
   public static a2 A2() { return new a2(); }  
  
   public static A1::C1 M1() { return new A1::C1(); }  
   public static A2::C2 M2() { return new A2::C2(); }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="c4e07-114">示例</span><span class="sxs-lookup"><span data-stu-id="c4e07-114">Example</span></span>

