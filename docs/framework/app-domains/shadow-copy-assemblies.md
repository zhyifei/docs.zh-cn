# <a name="shadow-copying-assemblies"></a><span data-ttu-id="eda7f-102">卷影复制程序集</span><span class="sxs-lookup"><span data-stu-id="eda7f-102">Shadow Copying Assemblies</span></span>

 <span data-ttu-id="eda7f-106">公共语言运行时会在加载程序集时锁定程序集文件，因此只有卸载此程序集才能更新此文件。</span><span class="sxs-lookup"><span data-stu-id="eda7f-106">The common language runtime locks an assembly file when the assembly is loaded, so the file cannot be updated until the assembly is unloaded.</span></span> <span data-ttu-id="eda7f-107">从应用程序域中卸载程序集的唯一方法是卸载应用程序域，因此在正常情况下，只有卸载了正在使用程序集的所有应用程序域才能在磁盘中更新此程序集。</span><span class="sxs-lookup"><span data-stu-id="eda7f-107">The only way to unload an assembly from an application domain is by unloading the application domain, so under normal circumstances, an assembly cannot be updated on disk until all the application domains that are using it have been unloaded.</span></span>  
  
 <span data-ttu-id="eda7f-108">应用程序域配置为卷影复制文件时，应用程序路径中的程序集复制到了另一个位置，并从此位置加载。</span><span class="sxs-lookup"><span data-stu-id="eda7f-108">When an application domain is configured to shadow copy files, assemblies from the application path are copied to another location and loaded from that location.</span></span> <span data-ttu-id="eda7f-109">此副本处于锁定状态，但原始程序集文件已解锁并可进行更新。</span><span class="sxs-lookup"><span data-stu-id="eda7f-109">The copy is locked, but the original assembly file is unlocked and can be updated.</span></span>  
  
> [!IMPORTANT]
>  <span data-ttu-id="eda7f-110">只有存储在应用程序目录或其子目录中的程序集才可进行卷影复制，这些程序集在配置应用程序域时由 <xref:System.AppDomainSetup.ApplicationBase%2A> 和 <xref:System.AppDomainSetup.PrivateBinPath%2A> 指定。</span><span class="sxs-lookup"><span data-stu-id="eda7f-110">The only assemblies that can be shadow copied are those stored in the application directory or its subdirectories, specified by the <xref:System.AppDomainSetup.ApplicationBase%2A> and <xref:System.AppDomainSetup.PrivateBinPath%2A> properties when the application domain is configured.</span></span> <span data-ttu-id="eda7f-111">存储在全局程序集缓存中的程序集不可进行卷影复制。</span><span class="sxs-lookup"><span data-stu-id="eda7f-111">Assemblies stored in the global assembly cache are not shadow copied.</span></span>

## <a name="startup-performance"></a><span data-ttu-id="eda7f-140">启动性能</span><span class="sxs-lookup"><span data-stu-id="eda7f-140">Startup Performance</span></span>

 <span data-ttu-id="eda7f-147">自 [!INCLUDE[net_v40_short](../../../includes/net-v40-short-md.md)] 以来，默认启动行为就是直接将应用程序目录中每个程序集的文件日期和时间与卷影复制目录中副本的进行比较。</span><span class="sxs-lookup"><span data-stu-id="eda7f-147">Beginning with the [!INCLUDE[net_v40_short](../../../includes/net-v40-short-md.md)], the default startup behavior is to directly compare the file date and time of each assembly in the application directory with the file date and time of the copy in the shadow copy directory.</span></span> <span data-ttu-id="eda7f-148">如果程序集已更新，则使用 .NET Framework 早期版本中的相同过程进行复制；否则，将加载卷影复制目录中的副本。</span><span class="sxs-lookup"><span data-stu-id="eda7f-148">If the assembly has been updated, it is copied by using the same procedure as in earlier versions of the .NET Framework; otherwise, the copy in the shadow copy directory is loaded.</span></span>  
  
 <span data-ttu-id="eda7f-149">对于其中程序集未频繁更改且通常程序集的较小子集发生更改的应用程序，产生的性能提升最大。</span><span class="sxs-lookup"><span data-stu-id="eda7f-149">The resulting performance improvement is largest for applications in which assemblies do not change frequently and changes usually occur in a small subset of assemblies.</span></span> <span data-ttu-id="eda7f-150">如果应用程序中的大部分程序集频繁发生更改，则新的默认行为可能导致性能退化。</span><span class="sxs-lookup"><span data-stu-id="eda7f-150">If a majority of assemblies in an application change frequently, the new default behavior might cause a performance regression.</span></span> <span data-ttu-id="eda7f-151">可以通过向配置文件添加 [\<shadowCopyVerifyByTimestamp> 元素](../../../docs/framework/configure-apps/file-schema/runtime/shadowcopyverifybytimestamp-element.md)来还原早期版本的 .NET Framework 的启动行为，其中 `enabled="false"`。</span><span class="sxs-lookup"><span data-stu-id="eda7f-151">You can restore the startup behavior of previous versions of the .NET Framework by adding the [\<shadowCopyVerifyByTimestamp> element](../../../docs/framework/configure-apps/file-schema/runtime/shadowcopyverifybytimestamp-element.md) to the configuration file, with `enabled="false"`.</span></span>
