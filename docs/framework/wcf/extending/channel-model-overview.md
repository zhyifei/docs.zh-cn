 <span data-ttu-id="2d6d2-112">首先，相似之处：在这两种情况下，堆栈中的每个层提供该层下面的层和公开该抽象仅向其上的一层的一些抽象。</span><span class="sxs-lookup"><span data-stu-id="2d6d2-112">First, the similarities: In both cases, each layer of the stack provides some abstraction of the world below that layer and exposed that abstraction only to the layer directly above it.</span></span> <span data-ttu-id="2d6d2-113">每一层只使用其下面一层的抽象。</span><span class="sxs-lookup"><span data-stu-id="2d6d2-113">Each layer uses the abstraction of only the layer directly below it.</span></span> <span data-ttu-id="2d6d2-114">而且在这两种情况下，两个堆栈通信时，每个层均与另一个堆栈中的相应层通信，例如，IP 层与 IP 层通信，TCP 层与 TCP 层通信，依此类推。</span><span class="sxs-lookup"><span data-stu-id="2d6d2-114">Also in both cases, when two stacks communicate, each layer communicates with the corresponding layer in the other stack, for example, the IP layer communicates with the IP layer and the TCP layer with the TCP layer, and so on.</span></span>

 <span data-ttu-id="2d6d2-130">另外，这些形状中的每个形状均有一个扩展 <xref:System.ServiceModel.Channels.ISessionChannel%601?displayProperty=nameWithType> 以支持会话的等效项。</span><span class="sxs-lookup"><span data-stu-id="2d6d2-130">Further, each of these shapes has an equivalent that extends <xref:System.ServiceModel.Channels.ISessionChannel%601?displayProperty=nameWithType> to support sessions.</span></span> <span data-ttu-id="2d6d2-131">这些是：</span><span class="sxs-lookup"><span data-stu-id="2d6d2-131">These are:</span></span>
