## <a name="concurrency"></a><span data-ttu-id="93a38-149">并发</span><span class="sxs-lookup"><span data-stu-id="93a38-149">Concurrency</span></span>  
 <span data-ttu-id="93a38-150">并发是对 <xref:System.ServiceModel.InstanceContext> 中在任一时刻处于活动状态的线程数量的控制。</span><span class="sxs-lookup"><span data-stu-id="93a38-150">Concurrency is the control of the number of threads active in an <xref:System.ServiceModel.InstanceContext> at any one time.</span></span> <span data-ttu-id="93a38-151">此控制是通过将 <xref:System.ServiceModel.ServiceBehaviorAttribute.ConcurrencyMode%2A?displayProperty=nameWithType> 与 <xref:System.ServiceModel.ConcurrencyMode> 枚举结合使用来实现的。</span><span class="sxs-lookup"><span data-stu-id="93a38-151">This is controlled by using the <xref:System.ServiceModel.ServiceBehaviorAttribute.ConcurrencyMode%2A?displayProperty=nameWithType> with the <xref:System.ServiceModel.ConcurrencyMode> enumeration.</span></span>  
  
 <span data-ttu-id="93a38-152">有以下三种可用的并发模式：</span><span class="sxs-lookup"><span data-stu-id="93a38-152">The following three concurrency modes are available:</span></span>  
  
- <span data-ttu-id="93a38-153"><xref:System.ServiceModel.ConcurrencyMode.Single>：每个实例上下文可拥有最多一次处理中的实例上下文的消息的其中一个线程。</span><span class="sxs-lookup"><span data-stu-id="93a38-153"><xref:System.ServiceModel.ConcurrencyMode.Single>: Each instance context is allowed to have a maximum of one thread processing messages in the instance context at a time.</span></span> <span data-ttu-id="93a38-154">其他希望使用同一个实例上下文的线程必须一直阻塞，直到原始线程退出该实例上下文为止。</span><span class="sxs-lookup"><span data-stu-id="93a38-154">Other threads wishing to use the same instance context must block until the original thread exits the instance context.</span></span>  
  
- <span data-ttu-id="93a38-155"><xref:System.ServiceModel.ConcurrencyMode.Multiple>：每个服务实例可以有多个线程同时处理消息。</span><span class="sxs-lookup"><span data-stu-id="93a38-155"><xref:System.ServiceModel.ConcurrencyMode.Multiple>: Each service instance can have multiple threads processing messages concurrently.</span></span> <span data-ttu-id="93a38-156">若要使用此并发模式，服务实现必须是线程安全的。</span><span class="sxs-lookup"><span data-stu-id="93a38-156">The service implementation must be thread-safe to use this concurrency mode.</span></span>  
  
- <span data-ttu-id="93a38-157"><xref:System.ServiceModel.ConcurrencyMode.Reentrant>：每个服务实例一次处理一个消息，但接受可重入操作的调用。</span><span class="sxs-lookup"><span data-stu-id="93a38-157"><xref:System.ServiceModel.ConcurrencyMode.Reentrant>: Each service instance processes one message at a time, but accepts re-entrant operation calls.</span></span> <span data-ttu-id="93a38-158">通过 WCF 客户端对象调用时，该服务仅接受这些调用。</span><span class="sxs-lookup"><span data-stu-id="93a38-158">The service only accepts these calls when it is calling out through a WCF client object.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="93a38-159">理解和开发能够安全地使用多个线程的代码可能比较困难。</span><span class="sxs-lookup"><span data-stu-id="93a38-159">Understanding and developing code that safely uses more than one thread can be difficult to write successfully.</span></span> <span data-ttu-id="93a38-160">在使用 <xref:System.ServiceModel.ConcurrencyMode.Multiple> 或 <xref:System.ServiceModel.ConcurrencyMode.Reentrant> 值之前，应确保已针对这些模式对服务进行了适当设计。</span><span class="sxs-lookup"><span data-stu-id="93a38-160">Before using <xref:System.ServiceModel.ConcurrencyMode.Multiple> or <xref:System.ServiceModel.ConcurrencyMode.Reentrant> values, ensure that your service is properly designed for these modes.</span></span> <span data-ttu-id="93a38-161">有关详细信息，请参阅 <xref:System.ServiceModel.ServiceBehaviorAttribute.ConcurrencyMode%2A>。</span><span class="sxs-lookup"><span data-stu-id="93a38-161">For more information, see <xref:System.ServiceModel.ServiceBehaviorAttribute.ConcurrencyMode%2A>.</span></span>  
  
 <span data-ttu-id="93a38-162">并发的使用与实例化模式有关。</span><span class="sxs-lookup"><span data-stu-id="93a38-162">The use of concurrency is related to the instancing mode.</span></span> <span data-ttu-id="93a38-163">在中<xref:System.ServiceModel.InstanceContextMode.PerCall>实例化过程中，并发没有关系，因为每个消息处理由一个新<xref:System.ServiceModel.InstanceContext>，因此，永远不会超过一个线程中处于活动状态<xref:System.ServiceModel.InstanceContext>。</span><span class="sxs-lookup"><span data-stu-id="93a38-163">In <xref:System.ServiceModel.InstanceContextMode.PerCall> instancing, concurrency is not relevant, because each message is processed by a new <xref:System.ServiceModel.InstanceContext> and, therefore, never more than one thread is active in the <xref:System.ServiceModel.InstanceContext>.</span></span>
