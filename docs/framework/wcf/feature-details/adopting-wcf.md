# <a name="adopting-windows-communication-foundation"></a><span data-ttu-id="d6b1c-102">采用 Windows Communication Foundation</span><span class="sxs-lookup"><span data-stu-id="d6b1c-102">Adopting Windows Communication Foundation</span></span>

<span data-ttu-id="d6b1c-103">您可以选择要用于新开发的 Windows Communication Foundation (WCF)，同时继续维护现有应用程序开发使用 ASP.NET。</span><span class="sxs-lookup"><span data-stu-id="d6b1c-103">You can choose to use Windows Communication Foundation (WCF) for new development, while continuing to maintain existing applications developed using ASP.NET.</span></span> <span data-ttu-id="d6b1c-104">因为 WCF 将是最合适的选择与在任何情况下.NET framework 构建应用程序之间的通信，所以它可以充当以解决各种软件通信问题的方式的标准工具 ASP.NET不能。</span><span class="sxs-lookup"><span data-stu-id="d6b1c-104">Because WCF is intended to be the most suitable choice for facilitating communication with applications built with the .NET Framework in any scenario, it can serve as a standard tool for solving a wide variety of software communications problems in a way that ASP.NET cannot.</span></span>

<span data-ttu-id="d6b1c-105">新的 WCF 应用程序可以部署在作为现有的 ASP.NET Web 服务在同一台计算机上。</span><span class="sxs-lookup"><span data-stu-id="d6b1c-105">New WCF applications can be deployed on the same machines as existing ASP.NET Web services.</span></span> <span data-ttu-id="d6b1c-106">如果现有的 ASP.NET Web 服务使用的.NET Framework 2.0 版之前的版本，然后可以使用 ASP.NET IIS 注册工具来有选择地将.NET Framework 2.0 部署到 IIS 应用程序是用来托管新的 WCF 应用程序。</span><span class="sxs-lookup"><span data-stu-id="d6b1c-106">If the existing ASP.NET Web services use a version of the .NET Framework prior to version 2.0, then you can use the ASP.NET IIS Registration Tool to selectively deploy the .NET Framework 2.0 to IIS applications in which new WCF applications are to be hosted.</span></span> <span data-ttu-id="d6b1c-107">该工具所述[ASP.NET IIS 注册工具 (Aspnet_regiis.exe)](https://go.microsoft.com/fwlink/?LinkId=94687)，并具有内置于 IIS 6.0 管理控制台用户界面。</span><span class="sxs-lookup"><span data-stu-id="d6b1c-107">That tool is documented at [ASP.NET IIS Registration Tool (Aspnet_regiis.exe)](https://go.microsoft.com/fwlink/?LinkId=94687), and has a user interface built into the IIS 6.0 management console.</span></span>

<span data-ttu-id="d6b1c-108">WCF 可以用于通过添加 WCF 服务配置为在 ASP.NET 兼容性模式下运行在 IIS 中的现有 ASP.NET Web 服务应用程序添加到现有的 ASP.NET Web 服务的新功能。</span><span class="sxs-lookup"><span data-stu-id="d6b1c-108">WCF can be used to add new features to existing ASP.NET Web services by adding WCF services configured to run in ASP.NET compatibility mode to existing ASP.NET Web service applications in IIS.</span></span> <span data-ttu-id="d6b1c-109">由于 ASP.NET 兼容模式下，新的 WCF 服务的代码可以访问和使用更新预先存在的 ASP.NET 代码，与相同的应用程序状态信息<xref:System.Web.HttpContext>类。</span><span class="sxs-lookup"><span data-stu-id="d6b1c-109">Because of ASP.NET compatibility mode, the code for the new WCF services can access and update the same application state information as the pre-existing ASP.NET code, by using the <xref:System.Web.HttpContext> class.</span></span> <span data-ttu-id="d6b1c-110">这些应用程序还可以共享相同的类库。</span><span class="sxs-lookup"><span data-stu-id="d6b1c-110">The applications can also share the same class libraries.</span></span>

<span data-ttu-id="d6b1c-111">WCF 客户端可以使用 ASP.NET Web 服务。</span><span class="sxs-lookup"><span data-stu-id="d6b1c-111">WCF clients can use ASP.NET Web services.</span></span> <span data-ttu-id="d6b1c-112">使用配置的 WCF 服务<xref:System.ServiceModel.BasicHttpBinding>可由 ASP.NET Web 服务客户端。</span><span class="sxs-lookup"><span data-stu-id="d6b1c-112">WCF services that are configured with the <xref:System.ServiceModel.BasicHttpBinding> can be used by ASP.NET Web service clients.</span></span> <span data-ttu-id="d6b1c-113">ASP.NET Web 服务可以与 WCF 应用程序共存，WCF 甚至可以用于将功能添加到现有的 ASP.NET Web 服务。</span><span class="sxs-lookup"><span data-stu-id="d6b1c-113">ASP.NET Web services can co-exist with WCF applications, and WCF can even be used to add features to existing ASP.NET Web services.</span></span> <span data-ttu-id="d6b1c-114">有了所有的 WCF 和 ASP.NET Web 服务可以一起使用这些方法，您可能想要将 ASP.NET Web 服务迁移到 WCF 中，仅当需要由 WCF 并不是 ASP.NET Web 服务提供的功能。</span><span class="sxs-lookup"><span data-stu-id="d6b1c-114">Given all of these ways in which WCF and ASP.NET Web services can be used together, you may want to migrate ASP.NET Web services to WCF only if you require features that are provided by WCF and not ASP.NET Web services.</span></span>

<span data-ttu-id="d6b1c-115">甚至在少数的情况下有必要，将代码从一种技术迁移到另一个很少是正确的方法。</span><span class="sxs-lookup"><span data-stu-id="d6b1c-115">Even in the few cases where it is necessary, migrating code from one technology to another is seldom the correct approach.</span></span> <span data-ttu-id="d6b1c-116">采用新技术的原因是为了满足早期技术无法满足的新需求。在上述情况下，正确的做法是设计一个新的解决方案来满足刚刚扩展的需求集。</span><span class="sxs-lookup"><span data-stu-id="d6b1c-116">The reason for adopting the new technology is to meet new requirements that cannot be met with the earlier technology, and in that case, the correct thing to do is to design a new solution to meet the newly-expanded set of requirements.</span></span> <span data-ttu-id="d6b1c-117">新的设计将受益于您对现有系统所拥有的经验以及您自该系统被设计以来获得的智慧。</span><span class="sxs-lookup"><span data-stu-id="d6b1c-117">The new design benefits from your experience with the existing system and from wisdom gained since that system was designed.</span></span> <span data-ttu-id="d6b1c-118">此外，新设计还可以利用新技术的所有功能，而不是在新平台上再现旧设计。</span><span class="sxs-lookup"><span data-stu-id="d6b1c-118">The new design can also use the full capabilities of the new technologies rather than reproducing the old design on the new platform.</span></span> <span data-ttu-id="d6b1c-119">在建立新设计的关键元素的原型之后，可以更轻松地在新系统中重用现有系统中的代码。</span><span class="sxs-lookup"><span data-stu-id="d6b1c-119">After prototyping key elements of the new design, it becomes easier to re-use code from the existing system within the new one.</span></span>
