# <a name="security-negotiation-and-timeouts"></a>安全协商和超时
如果客户端和服务进行身份验证，Windows Communication Foundation (WCF) 支持对服务凭据协商身份验证的一部分的其中一种模式。 在这种情况下，客户端和服务之间将进行潜在多路交换，以便将服务凭据传播到客户端。 <xref:System.ServiceModel.Channels.LocalServiceSecuritySettings.NegotiationTimeout%2A> 属性控制多路交换需要多长时间才能完成。 但此超时仅在交换实际经过的时间超过单个请求响应的时间时才适用。 如果协商在一次往返中完成，则该超时不适用。
