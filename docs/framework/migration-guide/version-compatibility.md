# <a name="version-compatibility-in-the-net-framework"></a><span data-ttu-id="041b0-102">.NET Framework 的版本兼容性</span><span class="sxs-lookup"><span data-stu-id="041b0-102">Version Compatibility in the .NET Framework</span></span>
<span data-ttu-id="041b0-103">向后兼容性表示为某个平台的特定版本开发的应用程序将在该平台的更高版本上运行。</span><span class="sxs-lookup"><span data-stu-id="041b0-103">Backward compatibility means that an app that was developed for a particular version of a platform will run on later versions of that platform.</span></span> <span data-ttu-id="041b0-104">.NET Framework 尝试最大程度地支持向后兼容性：为某个版本的 .NET Framework 编写的源代码应在更高版本的 .NET Framework 上编译，而在某个版本的 .NET Framework 上运行的二进制文件的行为方式应与其在更高版本的 .NET Framework 上的行为方式相同。</span><span class="sxs-lookup"><span data-stu-id="041b0-104">The .NET Framework tries to maximize backward compatibility: Source code written for one version of the .NET Framework should compile on later versions of the .NET Framework, and binaries that run on one version of the .NET Framework should behave identically on later versions of the .NET Framework.</span></span>  
  
<a name="Apps"></a>   
## <a name="version-compatibility-for-apps"></a><span data-ttu-id="041b0-105">应用程序的版本兼容性</span><span class="sxs-lookup"><span data-stu-id="041b0-105">Version compatibility for apps</span></span>  
 <span data-ttu-id="041b0-106">默认情况下，应用将在其目标 .NET Framework 版本上运行。</span><span class="sxs-lookup"><span data-stu-id="041b0-106">By default, an app runs on the version of the .NET Framework that it was built for.</span></span> <span data-ttu-id="041b0-107">如果该版本不存在且应用程序配置文件未定义支持的版本，则可能出现 .NET Framework 初始化错误。</span><span class="sxs-lookup"><span data-stu-id="041b0-107">If that version is not present and the app configuration file does not define supported versions, a .NET Framework initialization error may occur.</span></span> <span data-ttu-id="041b0-108">在此情况下，尝试运行应用程序将失败。</span><span class="sxs-lookup"><span data-stu-id="041b0-108">In this case, the attempt to run the app will fail.</span></span>  
  
 <span data-ttu-id="041b0-109">若要定义运行应用的特定版本，请将一个或多个 [\<supportedRuntime>](../../../docs/framework/configure-apps/file-schema/startup/supportedruntime-element.md) 元素添加到应用的配置文件中。</span><span class="sxs-lookup"><span data-stu-id="041b0-109">To define the specific versions on which your app runs, add one or more [\<supportedRuntime>](../../../docs/framework/configure-apps/file-schema/startup/supportedruntime-element.md) elements to your app's configuration file.</span></span> <span data-ttu-id="041b0-110">每个 `<supportedRuntime>` 元素都列出了支持的运行时版本，第一个元素指定了优先级最高的版本，最后一个元素指定了优先级最低的版本。</span><span class="sxs-lookup"><span data-stu-id="041b0-110">Each `<supportedRuntime>` element lists a supported version of the runtime, with the first specifying the most preferred version and the last specifying the least preferred version.</span></span>  
  
```xml  
<configuration>  
   <startup>  
      <supportedRuntime version="v2.0.50727" />  
      <supportedRuntime version="v4.0" />  
   </startup>  
</configuration>  
```  
  
 <span data-ttu-id="041b0-111">有关详细信息，请参阅[如何：配置应用以支持 .NET Framework 4 或 4.x](../../../docs/framework/migration-guide/how-to-configure-an-app-to-support-net-framework-4-or-4-5.md)。</span><span class="sxs-lookup"><span data-stu-id="041b0-111">For more information, see [How to: Configure an App to Support .NET Framework 4 or 4.x](../../../docs/framework/migration-guide/how-to-configure-an-app-to-support-net-framework-4-or-4-5.md).</span></span>  
  
## <a name="version-compatibility-for-components"></a><span data-ttu-id="041b0-112">组件的版本兼容性</span><span class="sxs-lookup"><span data-stu-id="041b0-112">Version compatibility for components</span></span>  
 <span data-ttu-id="041b0-113">应用程序可控制运行它的 .NET Framework 版本，但组件不能。</span><span class="sxs-lookup"><span data-stu-id="041b0-113">An app can control the version of the .NET Framework on which it runs, but a component cannot.</span></span> <span data-ttu-id="041b0-114">由于组件和类库在特定应用的上下文中加载，因此它们会自动在运行应用的 .NET Framework 版本上运行。</span><span class="sxs-lookup"><span data-stu-id="041b0-114">Components and class libraries are loaded in the context of a particular app, and therefore automatically run on the version of the .NET Framework that the app runs on.</span></span>  
  
 <span data-ttu-id="041b0-115">由于存在此限制，因此兼容性保证对组件特别重要。</span><span class="sxs-lookup"><span data-stu-id="041b0-115">Because of this restriction, compatibility guarantees are especially important for components.</span></span> <span data-ttu-id="041b0-116">从 .NET Framework 4 开始，你可以通过将 <xref:System.Runtime.Versioning.ComponentGuaranteesAttribute?displayProperty=nameWithType> 特性应用于某个组件，来指定希望该组件与多个版本的兼容程度。</span><span class="sxs-lookup"><span data-stu-id="041b0-116">Starting with the .NET Framework 4, you can specify the degree to which a component is expected to remain compatible across multiple versions by applying the <xref:System.Runtime.Versioning.ComponentGuaranteesAttribute?displayProperty=nameWithType> attribute to that component.</span></span> <span data-ttu-id="041b0-117">工具可使用此特性来检测组件的将来版本中的兼容性保证的潜在冲突。</span><span class="sxs-lookup"><span data-stu-id="041b0-117">Tools can use this attribute to detect potential violations of the compatibility guarantee in future versions of a component.</span></span>  
  
## <a name="backward-compatibility-and-the-net-framework-45"></a><span data-ttu-id="041b0-118">向后兼容性和 .NET Framework 4.5</span><span class="sxs-lookup"><span data-stu-id="041b0-118">Backward compatibility and the .NET Framework 4.5</span></span>  
 <span data-ttu-id="041b0-119">.NET Framework 4.5 及更高版本与使用早期版本的 .NET Framework 生成的应用向后兼容。</span><span class="sxs-lookup"><span data-stu-id="041b0-119">The .NET Framework 4.5 and later versions are backward-compatible with apps that were built with earlier versions of the .NET Framework.</span></span> <span data-ttu-id="041b0-120">换句话说，使用早期版本的 .NET Framework 生成的应用和组件将运行，而无需在 .NET Framework 4.5 及更高版本上进行修改。</span><span class="sxs-lookup"><span data-stu-id="041b0-120">In other words, apps and components built with previous versions will work without modification on the .NET Framework 4.5 and later versions.</span></span> <span data-ttu-id="041b0-121">但在默认情况下，应用在其进行开发的公共语言运行时版本上运行，因此必须提供配置文件，使应用能在 .NET Framework 4.5 及更高版本上运行。</span><span class="sxs-lookup"><span data-stu-id="041b0-121">However, by default, apps run on the version of the common language runtime for which they were developed, so you may have to provide a configuration file to enable your app to run on the .NET Framework 4.5 or later versions.</span></span> <span data-ttu-id="041b0-122">有关详细信息，请参阅本文前面的[应用的版本兼容性](#Apps)一节。</span><span class="sxs-lookup"><span data-stu-id="041b0-122">For more information, see the [Version compatibility for apps](#Apps) section earlier in this article.</span></span>  
  
 <span data-ttu-id="041b0-123">实际上，.NET Framework 中看似无关紧要的更改和编程技术上的更改会损坏此兼容性。</span><span class="sxs-lookup"><span data-stu-id="041b0-123">In practice, this compatibility can be broken by seemingly inconsequential changes in the .NET Framework and changes in programming techniques.</span></span> <span data-ttu-id="041b0-124">例如，.NET Framework 4.5 中的性能改进会公开早期版本中未出现的争用条件。</span><span class="sxs-lookup"><span data-stu-id="041b0-124">For example, performance improvements in the .NET Framework 4.5 can expose a race condition that did not occur on earlier versions.</span></span> <span data-ttu-id="041b0-125">同样，使用 .NET Framework 程序集的硬编码路径，执行与特定版本的 .NET Framework 的相等比较，以及使用反射获取私有字段的值都不是向后兼容的做法。</span><span class="sxs-lookup"><span data-stu-id="041b0-125">Similarly, using a hard-coded path to .NET Framework assemblies, performing an equality comparison with a particular version of the .NET Framework, and getting the value of a private field by using reflection are not backward-compatible practices.</span></span> <span data-ttu-id="041b0-126">此外，每个版本的 .NET Framework 都包含 Bug 修复和可能影响某些应用程序和组件的兼容性的安全相关更改。</span><span class="sxs-lookup"><span data-stu-id="041b0-126">In addition, each version of the .NET Framework includes bug fixes and security-related changes that can affect the compatibility of some apps and components.</span></span>
