# <a name="entity-data-model-key-concepts"></a><span data-ttu-id="50e2d-102">实体数据模型关键概念</span><span class="sxs-lookup"><span data-stu-id="50e2d-102">Entity Data Model Key Concepts</span></span>
<span data-ttu-id="50e2d-103">实体数据模型（EDM）使用三个关键概念来描述数据结构：*实体类型*、*关联类型*和*属性*。</span><span class="sxs-lookup"><span data-stu-id="50e2d-103">The Entity Data Model (EDM) uses three key concepts to describe the structure of data: *entity type*, *association type*, and *property*.</span></span> <span data-ttu-id="50e2d-104">在任何 EDM 实现中，这些都是描述数据结构的最重要概念。</span><span class="sxs-lookup"><span data-stu-id="50e2d-104">These are the most important concepts in describing the structure of data in any implementation of the EDM.</span></span>  
  
## <a name="entity-type"></a><span data-ttu-id="50e2d-105">实体类型</span><span class="sxs-lookup"><span data-stu-id="50e2d-105">Entity Type</span></span>  
 <span data-ttu-id="50e2d-106">[实体类型](entity-type.md)是用于描述实体数据模型的数据结构的基本构造块。</span><span class="sxs-lookup"><span data-stu-id="50e2d-106">The [entity type](entity-type.md) is the fundamental building block for describing the structure of data with the Entity Data Model.</span></span> <span data-ttu-id="50e2d-107">在概念模型中，实体类型是从[属性](property.md)构造的，描述顶级概念的结构，如业务应用程序中的客户和订单。</span><span class="sxs-lookup"><span data-stu-id="50e2d-107">In a conceptual model, entity types are constructed from [properties](property.md) and describe the structure of top-level concepts, such as a customers and orders in a business application.</span></span> <span data-ttu-id="50e2d-108">正如计算机程序中的类定义是用于类实例的模板一样，实体类型也是实体的模板。</span><span class="sxs-lookup"><span data-stu-id="50e2d-108">In the same way that a class definition in a computer program is a template for instances of the class, an entity type is a template for entities.</span></span> <span data-ttu-id="50e2d-109">实体表示一个特定对象（例如特定的客户或订单）。</span><span class="sxs-lookup"><span data-stu-id="50e2d-109">An entity represents a specific object (such as a specific customer or order).</span></span> <span data-ttu-id="50e2d-110">每个实体都必须在[实体集中](entity-set.md)具有唯一的[实体键](entity-key.md)。</span><span class="sxs-lookup"><span data-stu-id="50e2d-110">Each entity must have a unique [entity key](entity-key.md) within an [entity set](entity-set.md).</span></span>  <span data-ttu-id="50e2d-111">实体集是指特定实体类型的实例集合。</span><span class="sxs-lookup"><span data-stu-id="50e2d-111">An entity set is a collection of instances of a specific entity type.</span></span> <span data-ttu-id="50e2d-112">实体集（和[关联集](association-set.md)）按逻辑分组在[实体容器](entity-container.md)中。</span><span class="sxs-lookup"><span data-stu-id="50e2d-112">Entity sets (and [association sets](association-set.md)) are logically grouped in an [entity container](entity-container.md).</span></span>  
  
 <span data-ttu-id="50e2d-113">实体类型支持继承功能：即一个实体类型可以从另一个实体类型中派生。</span><span class="sxs-lookup"><span data-stu-id="50e2d-113">Inheritance is supported with entity types: that is, one entity type can be derived from another.</span></span> <span data-ttu-id="50e2d-114">有关详细信息，请[参阅实体数据模型：继承](entity-data-model-inheritance.md)。</span><span class="sxs-lookup"><span data-stu-id="50e2d-114">For more information, see [Entity Data Model: Inheritance](entity-data-model-inheritance.md).</span></span>  
  
## <a name="association-type"></a><span data-ttu-id="50e2d-115">关联类型</span><span class="sxs-lookup"><span data-stu-id="50e2d-115">Association Type</span></span>  
 <span data-ttu-id="50e2d-116">[关联类型](association-type.md)（也称为关联）是用于描述实体数据模型中的关系的基本构造块。</span><span class="sxs-lookup"><span data-stu-id="50e2d-116">An [association type](association-type.md) (also called an association) is the fundamental building block for describing relationships in the Entity Data Model.</span></span> <span data-ttu-id="50e2d-117">在概念模型中，关联表示两个实体类型（例如 Customer 和 Order）之间的关系。</span><span class="sxs-lookup"><span data-stu-id="50e2d-117">In a conceptual model, an association represents a relationship between two entity types (such as Customer and Order).</span></span> <span data-ttu-id="50e2d-118">每个关联都有两个[关联端](association-end.md)，用于指定关联中涉及的实体类型。</span><span class="sxs-lookup"><span data-stu-id="50e2d-118">Every association has two [association ends](association-end.md) that specify the entity types involved in the association.</span></span> <span data-ttu-id="50e2d-119">每个关联端还指定一个[关联端多重性](association-end-multiplicity.md)，以指示可以在该关联端的实体数量。</span><span class="sxs-lookup"><span data-stu-id="50e2d-119">Each association end also specifies an [association end multiplicity](association-end-multiplicity.md) that indicates the number of entities that can be at that end of the association.</span></span> <span data-ttu-id="50e2d-120">Association 端重数的值可以为一（1）、零或一（0 ..1）或多（\*）。</span><span class="sxs-lookup"><span data-stu-id="50e2d-120">An association end multiplicity can have a value of one (1), zero or one (0..1), or many (\*).</span></span> <span data-ttu-id="50e2d-121">可以通过[导航属性](navigation-property.md)访问关联一端的实体; 如果在实体类型上公开实体，则可以通过外键访问。</span><span class="sxs-lookup"><span data-stu-id="50e2d-121">Entities at one end of an association can be accessed through [navigation properties](navigation-property.md), or through foreign keys if they are exposed on an entity type.</span></span> <span data-ttu-id="50e2d-122">有关详细信息，请参阅[外键属性](foreign-key-property.md)。</span><span class="sxs-lookup"><span data-stu-id="50e2d-122">For more information, see [foreign key property](foreign-key-property.md).</span></span>  
  
 <span data-ttu-id="50e2d-123">在应用程序中，关联实例表示特定的关联（例如 Customer 实例与 Order 实例之间的关联）。</span><span class="sxs-lookup"><span data-stu-id="50e2d-123">In an application, an instance of an association represents a specific association (such as an association between an instance of Customer and instances of Order).</span></span> <span data-ttu-id="50e2d-124">关联实例按逻辑分组到一个[关联集](association-set.md)。</span><span class="sxs-lookup"><span data-stu-id="50e2d-124">Association instances are logically grouped in an [association set](association-set.md).</span></span> <span data-ttu-id="50e2d-125">关联集（和[实体集](entity-set.md)）按逻辑分组在[实体容器](entity-container.md)中。</span><span class="sxs-lookup"><span data-stu-id="50e2d-125">Association sets (and [entity sets](entity-set.md)) are logically grouped in an [entity container](entity-container.md).</span></span>  
  
## <a name="property"></a><span data-ttu-id="50e2d-126">属性</span><span class="sxs-lookup"><span data-stu-id="50e2d-126">Property</span></span>  
 <span data-ttu-id="50e2d-127">[实体类型](entity-type.md)包含定义其结构和特征的[属性](property.md)。</span><span class="sxs-lookup"><span data-stu-id="50e2d-127">[Entity types](entity-type.md) contain [properties](property.md) that define their structure and characteristics.</span></span> <span data-ttu-id="50e2d-128">例如，Customer 实体类型可能具有 CustomerId、Name 及 Address 这样的属性。</span><span class="sxs-lookup"><span data-stu-id="50e2d-128">For example, a Customer entity type may have properties such as CustomerId, Name, and Address.</span></span>  
  
 <span data-ttu-id="50e2d-129">概念模型中的属性类似于对计算机程序中的类所定义的属性。</span><span class="sxs-lookup"><span data-stu-id="50e2d-129">Properties in a conceptual model are analogous to properties defined on a class in a computer program.</span></span> <span data-ttu-id="50e2d-130">正如类的属性定义类的形状和携带有关对象的信息一样，概念模型中的属性也定义实体类型的形状和携带有关实体类型实例的信息。</span><span class="sxs-lookup"><span data-stu-id="50e2d-130">In the same way that properties on a class define the shape of the class and carry information about objects, properties in a conceptual model define the shape of an entity type and carry information about entity type instances.</span></span>  
  
 <span data-ttu-id="50e2d-131">属性可以包含基元数据（例如字符串、整数或布尔值）或结构化数据（例如复杂类型）。</span><span class="sxs-lookup"><span data-stu-id="50e2d-131">A property can contain primitive data (such as a string, an integer, or a Boolean value), or structured data (such as a complex type).</span></span> <span data-ttu-id="50e2d-132">有关详细信息，请[参阅实体数据模型：基元数据类型](entity-data-model-primitive-data-types.md)。</span><span class="sxs-lookup"><span data-stu-id="50e2d-132">For more information, see [Entity Data Model: Primitive Data Types](entity-data-model-primitive-data-types.md).</span></span>
