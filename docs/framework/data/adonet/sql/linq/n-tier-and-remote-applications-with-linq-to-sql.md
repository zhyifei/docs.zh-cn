# <a name="n-tier-and-remote-applications-with-linq-to-sql"></a><span data-ttu-id="6d2aa-102">使用 LINQ to SQL 的 N 层和远程应用程序</span><span class="sxs-lookup"><span data-stu-id="6d2aa-102">N-Tier and Remote Applications with LINQ to SQL</span></span>
<span data-ttu-id="6d2aa-103">可以创建使用 [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] 的 n 层或多层应用程序。</span><span class="sxs-lookup"><span data-stu-id="6d2aa-103">You can create n-tier or multitier applications that use [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)].</span></span> <span data-ttu-id="6d2aa-104">通常情况下，[!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)]数据上下文、 实体类和查询构造逻辑位于中间层上作为数据访问层 (DAL)。</span><span class="sxs-lookup"><span data-stu-id="6d2aa-104">Typically, the [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] data context, entity classes, and query construction logic are located on the middle tier as the data access layer (DAL).</span></span> <span data-ttu-id="6d2aa-105">业务逻辑和任何非持久性数据都可以在实体的分部类和分部方法中以及数据上下文中完整实现，也可以在单独的类中实现。</span><span class="sxs-lookup"><span data-stu-id="6d2aa-105">Business logic and any non-persistent data can be implemented completely in partial classes and methods of entities and the data context, or it can be implemented in separate classes.</span></span>

 <span data-ttu-id="6d2aa-106">客户端或表示层调用中间层的远程接口上的方法，而该层上的 DAL 会执行映射到 <xref:System.Data.Linq.DataContext> 方法的查询或存储过程。</span><span class="sxs-lookup"><span data-stu-id="6d2aa-106">The client or presentation layer calls methods on the middle-tier's remote interface, and the DAL on that tier will execute queries or stored procedures that are mapped to <xref:System.Data.Linq.DataContext> methods.</span></span> <span data-ttu-id="6d2aa-107">中间层通常以实体或代理对象的 XML 表示形式向客户端返回数据。</span><span class="sxs-lookup"><span data-stu-id="6d2aa-107">The middle tier returns the data to clients typically as XML representations of entities or proxy objects.</span></span>

 <span data-ttu-id="6d2aa-108">在中间层上，实体由数据上下文创建，数据上下文会跟踪实体的状态，并管理从数据库延迟加载实体以及将更改提交给数据库的过程。</span><span class="sxs-lookup"><span data-stu-id="6d2aa-108">On the middle tier, entities are created by the data context, which tracks their state, and manages deferred loading from and submission of changes to the database.</span></span> <span data-ttu-id="6d2aa-109">这些实体被“附加”到 `DataContext`。</span><span class="sxs-lookup"><span data-stu-id="6d2aa-109">These entities are "attached" to the `DataContext`.</span></span> <span data-ttu-id="6d2aa-110">但是，在实体通过序列化发送到其他层后，这些实体会分离出来，这意味着 `DataContext` 不再跟踪它们的状态。</span><span class="sxs-lookup"><span data-stu-id="6d2aa-110">However, after the entities are sent to another tier through serialization, they become detached, which means the `DataContext` is no longer tracking their state.</span></span> <span data-ttu-id="6d2aa-111">必须在将客户端发送回来进行更新的实体重新附加到数据上下文之后，[!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] 才能将更改提交给数据库。</span><span class="sxs-lookup"><span data-stu-id="6d2aa-111">Entities that the client sends back for updates must be reattached to the data context before [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] can submit the changes to the database.</span></span> <span data-ttu-id="6d2aa-112">如果原始值和/或时间戳是开放式并发检查所必需的，则客户端负责将它们返回给中间层。</span><span class="sxs-lookup"><span data-stu-id="6d2aa-112">The client is responsible for providing original values and/or timestamps back to the middle tier if those are required for optimistic concurrency checks.</span></span>

 <span data-ttu-id="6d2aa-113">在 ASP.NET 应用程序中，<xref:System.Web.UI.WebControls.LinqDataSource> 管理这一复杂过程的大部分工作。</span><span class="sxs-lookup"><span data-stu-id="6d2aa-113">In ASP.NET applications, the <xref:System.Web.UI.WebControls.LinqDataSource> manages most of this complexity.</span></span> <span data-ttu-id="6d2aa-114">有关详细信息，请参阅[NIB: LinqDataSource Web 服务器控件概述](https://msdn.microsoft.com/library/104cfc3f-7385-47d3-8a51-830dfa791136)。</span><span class="sxs-lookup"><span data-stu-id="6d2aa-114">For more information, see [NIB: LinqDataSource Web Server Control Overview](https://msdn.microsoft.com/library/104cfc3f-7385-47d3-8a51-830dfa791136).</span></span>
