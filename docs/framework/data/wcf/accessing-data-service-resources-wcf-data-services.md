## <a name="addressing-resources"></a><span data-ttu-id="2abb4-107">处理资源</span><span class="sxs-lookup"><span data-stu-id="2abb4-107">Addressing Resources</span></span>  
 <span data-ttu-id="2abb4-108">在 [!INCLUDE[ssODataShort](../../../../includes/ssodatashort-md.md)] 中，您可以通过使用 URI 对数据模型公开的任何数据进行寻址。</span><span class="sxs-lookup"><span data-stu-id="2abb4-108">In [!INCLUDE[ssODataShort](../../../../includes/ssodatashort-md.md)], you address any data exposed by the data model by using a URI.</span></span> <span data-ttu-id="2abb4-109">例如，下面的 URI 返回 Customers 实体集包含的所有实例的 Customer 实体类型的项的源：</span><span class="sxs-lookup"><span data-stu-id="2abb4-109">For example, the following URI returns a feed that is the Customers entity set, which contains entries for all instances of the Customer entity type:</span></span>  
  
```  
http://services.odata.org/Northwind/Northwind.svc/Customers  
```  
  
 <span data-ttu-id="2abb4-110">实体具有称为实体键的特殊属性。</span><span class="sxs-lookup"><span data-stu-id="2abb4-110">Entities have special properties called entity keys.</span></span> <span data-ttu-id="2abb4-111">实体键用于在实体集中唯一标识某个实体。</span><span class="sxs-lookup"><span data-stu-id="2abb4-111">An entity key is used to uniquely identify a single entity in an entity set.</span></span> <span data-ttu-id="2abb4-112">这样，您可以在实体集中对某种实体类型的特定实例进行寻址。</span><span class="sxs-lookup"><span data-stu-id="2abb4-112">This enables you to address a specific instance of an entity type in the entity set.</span></span> <span data-ttu-id="2abb4-113">例如，下面的 URI 返回 Customer 实体类型的具有键值 `ALFKI` 的特定实例的项：</span><span class="sxs-lookup"><span data-stu-id="2abb4-113">For example, the following URI returns an entry for a specific instance of the Customer entity type that has a key value of `ALFKI`:</span></span>  
  
```  
http://services.odata.org/Northwind/Northwind.svc/Customers('ALFKI')  
```  
  
 <span data-ttu-id="2abb4-114">也可以对实体实例的基元属性和复杂属性进行单独寻址。</span><span class="sxs-lookup"><span data-stu-id="2abb4-114">Primitive and complex properties of an entity instance can also be individually addressed.</span></span> <span data-ttu-id="2abb4-115">例如，下面的 URI 返回一个包含特定客户的 `ContactName` 属性值的 XML 元素：</span><span class="sxs-lookup"><span data-stu-id="2abb4-115">For example, the following URI returns an XML element that contains the `ContactName` property value for a specific Customer:</span></span>

http://services.odata.org/Northwind/Northwind.svc/Customers('ALFKI')/ContactName  
```  
  
 <span data-ttu-id="2abb4-116">如果在上面的 URI 中包括 `$value` 终结点，则只在响应消息中返回基元属性的值。</span><span class="sxs-lookup"><span data-stu-id="2abb4-116">When you include the `$value` endpoint in the previous URI, only the value of the primitive property is returned in the response message.</span></span> <span data-ttu-id="2abb4-117">下面的示例只返回字符串“Maria Anders”，而不返回 XML 元素：</span><span class="sxs-lookup"><span data-stu-id="2abb4-117">The following example returns only the string "Maria Anders" without the XML element:</span></span>

```  
  
 <span data-ttu-id="2abb4-118">实体之间的关系在数据模型中由关联定义。</span><span class="sxs-lookup"><span data-stu-id="2abb4-118">Relationships between entities are defined in the data model by associations.</span></span> <span data-ttu-id="2abb4-119">通过这些关联，可以使用实体实例的导航属性对相关实体进行寻址。</span><span class="sxs-lookup"><span data-stu-id="2abb4-119">These associations enable you to address related entities by using navigation properties of an entity instance.</span></span> <span data-ttu-id="2abb4-120">对于多对一的关系，导航属性可返回单个相关实体；对于一对多的关系，导航属性可返回一组相关实体。</span><span class="sxs-lookup"><span data-stu-id="2abb4-120">A navigation property can return either a single related entity, in the case of a many-to-one relationship, or a set of related entities, in the case of a one-to-many relationship.</span></span> <span data-ttu-id="2abb4-121">例如，下面的 URI 返回一个作为与特定客户相关的所有订单集的源：</span><span class="sxs-lookup"><span data-stu-id="2abb4-121">For example, the following URI returns a feed that is the set of all the Orders that are related to a specific Customer:</span></span>  
  
```  
http://services.odata.org/Northwind/Northwind.svc/Customers('ALFKI')/Orders  
```  
  
 <span data-ttu-id="2abb4-122">通常为双向的关系由一对导航属性表示。</span><span class="sxs-lookup"><span data-stu-id="2abb4-122">Relationships, which are usually bi-directional, are represented by a pair of navigation properties.</span></span> <span data-ttu-id="2abb4-123">作为对前一示例中所示的关系的反转，下面的 URI 返回对特定 Order 实体所属的 Customer 实体的引用：</span><span class="sxs-lookup"><span data-stu-id="2abb4-123">As the reverse of the relationship shown in the previous example, the following URI returns a reference to the Customer entity to which a specific Order entity belongs:</span></span>

```  
  
 [!INCLUDE[ssODataShort](../../../../includes/ssodatashort-md.md)]<span data-ttu-id="2abb4-124"> 此外可以基于查询表达式的结果进行资源寻址。</span><span class="sxs-lookup"><span data-stu-id="2abb4-124"> also enables you to address resources based on the results of query expressions.</span></span> <span data-ttu-id="2abb4-125">这样，可以基于计算的表达式的资源集进行筛选。</span><span class="sxs-lookup"><span data-stu-id="2abb4-125">This makes it possible to filter sets of resources based on an evaluated expression.</span></span> <span data-ttu-id="2abb4-126">例如，下面的 URI 对资源进行筛选以仅返回指定客户自 1997 年 9 月 22 日起已发货的订单：</span><span class="sxs-lookup"><span data-stu-id="2abb4-126">For example, the following URI filters the resources to return only the Orders for the specified Customer that have shipped since September 22, 1997:</span></span>

```  
  
 <span data-ttu-id="2abb4-127">有关详细信息，请参阅[OData: URI 约定](https://go.microsoft.com/fwlink/?LinkId=185564)。</span><span class="sxs-lookup"><span data-stu-id="2abb4-127">For more information, see [OData: URI Conventions](https://go.microsoft.com/fwlink/?LinkId=185564).</span></span>
