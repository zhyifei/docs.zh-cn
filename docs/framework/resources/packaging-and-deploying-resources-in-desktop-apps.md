# <a name="packaging-and-deploying-resources-in-desktop-apps"></a><span data-ttu-id="20cb0-102">打包和部署桌面应用程序中的资源</span><span class="sxs-lookup"><span data-stu-id="20cb0-102">Packaging and Deploying Resources in Desktop Apps</span></span>
<span data-ttu-id="20cb0-103">应用程序依靠 .NET Framework Resource Manager（由 <xref:System.Resources.ResourceManager> 类表示）来检索已本地化的资源。</span><span class="sxs-lookup"><span data-stu-id="20cb0-103">Applications rely on the .NET Framework Resource Manager, represented by the <xref:System.Resources.ResourceManager> class, to retrieve localized resources.</span></span> <span data-ttu-id="20cb0-104">Resource Manager 假定使用中枢轮辐式模型来打包和部署资源。</span><span class="sxs-lookup"><span data-stu-id="20cb0-104">The Resource Manager assumes that a hub and spoke model is used to package and deploy resources.</span></span> <span data-ttu-id="20cb0-105">中枢即主程序集，包含不可本地化的可执行代码和单个区域性（称作非特定区域性或默认区域性）的资源。</span><span class="sxs-lookup"><span data-stu-id="20cb0-105">The hub is the main assembly that contains the nonlocalizable executable code and the resources for a single culture, called the neutral or default culture.</span></span> <span data-ttu-id="20cb0-106">默认区域性是应用程序的回退区域性；如果找不到已本地化的资源，则使用默认区域性的资源。</span><span class="sxs-lookup"><span data-stu-id="20cb0-106">The default culture is the fallback culture for the application; it is the culture whose resources are used if localized resources cannot be found.</span></span> <span data-ttu-id="20cb0-107">每条轮辐均连接到一个附属程序集，该附属程序集包含单个区域性的资源，但不包含任何代码。</span><span class="sxs-lookup"><span data-stu-id="20cb0-107">Each spoke connects to a satellite assembly that contains the resources for a single culture, but does not contain any code.</span></span>  
  
 <span data-ttu-id="20cb0-108">此模型有以下优点：</span><span class="sxs-lookup"><span data-stu-id="20cb0-108">There are several advantages to this model:</span></span>  
  
-   <span data-ttu-id="20cb0-109">部署应用程序后，可针对新区域性以增量方式添加资源。</span><span class="sxs-lookup"><span data-stu-id="20cb0-109">You can incrementally add resources for new cultures after you have deployed an application.</span></span> <span data-ttu-id="20cb0-110">因为特定于区域性资源的后续开发可能需要大量时间，这使你能够先发布主要应用程序，并在以后发布特定于区域性的资源。</span><span class="sxs-lookup"><span data-stu-id="20cb0-110">Because subsequent development of culture-specific resources can require a significant amount of time, this allows you to release your main application first, and deliver culture-specific resources at a later date.</span></span>  
  
-   <span data-ttu-id="20cb0-111">可更新和更改应用程序的附属程序集，无需重新编译应用程序。</span><span class="sxs-lookup"><span data-stu-id="20cb0-111">You can update and change an application's satellite assemblies without recompiling the application.</span></span>  
  
-   <span data-ttu-id="20cb0-112">应用程序只需加载那些包含特定区域性所需资源的附属程序集。</span><span class="sxs-lookup"><span data-stu-id="20cb0-112">An application needs to load only those satellite assemblies that contain the resources needed for a particular culture.</span></span> <span data-ttu-id="20cb0-113">这可显著减少系统资源的使用。</span><span class="sxs-lookup"><span data-stu-id="20cb0-113">This can significantly reduce the use of system resources.</span></span>  
  
 <span data-ttu-id="20cb0-114">但是，该模型也有以下缺点：</span><span class="sxs-lookup"><span data-stu-id="20cb0-114">However, there are also disadvantages to this model:</span></span>  
  
-   <span data-ttu-id="20cb0-115">必须管理多组资源。</span><span class="sxs-lookup"><span data-stu-id="20cb0-115">You must manage multiple sets of resources.</span></span>  
  
-   <span data-ttu-id="20cb0-116">测试应用程序的初始成本增加，因为必须测试多种配置。</span><span class="sxs-lookup"><span data-stu-id="20cb0-116">The initial cost of testing an application increases, because you must test several configurations.</span></span> <span data-ttu-id="20cb0-117">请注意，从长远角度看，测试使用多个附属程序集测试多个核心应用程序会比测试并维护多个并行国际版本更轻松，成本更低。</span><span class="sxs-lookup"><span data-stu-id="20cb0-117">Note that in the long term it will be easier and less expensive to test one core application with several satellites, than to test and maintain several parallel international versions.</span></span>  
  
## <a name="resource-naming-conventions"></a><span data-ttu-id="20cb0-118">资源命名约定</span><span class="sxs-lookup"><span data-stu-id="20cb0-118">Resource Naming Conventions</span></span>

## <a name="the-resource-fallback-process"></a><span data-ttu-id="20cb0-126">资源回退进程</span><span class="sxs-lookup"><span data-stu-id="20cb0-126">The Resource Fallback Process</span></span>  
 <span data-ttu-id="20cb0-127">用于打包和部署资源的中枢轮辐式模型可使用回退过程来定位相应的资源。</span><span class="sxs-lookup"><span data-stu-id="20cb0-127">The hub and spoke model for packaging and deploying resources uses a fallback process to locate appropriate resources.</span></span> <span data-ttu-id="20cb0-128">如果应用程序请求不可用的已本地化资源，则公共语言运行时会在区域性层次结构中查找与用户应用程序请求最匹配的合适回退资源，并且只在最后迫不得已的情况下才引发异常。</span><span class="sxs-lookup"><span data-stu-id="20cb0-128">If an application requests a localized resource  that is unavailable, the common language runtime searches the hierarchy of cultures for an appropriate fallback resource that most closely matches the user's application's request, and throws an exception only as a last resort.</span></span> <span data-ttu-id="20cb0-129">如果在层次结构的任意级别找到了合适的资源，运行时就会使用该资源。</span><span class="sxs-lookup"><span data-stu-id="20cb0-129">At each level of the hierarchy, if an appropriate resource is found, the runtime uses it.</span></span> <span data-ttu-id="20cb0-130">如果未找到资源，则将继续在下一级别中搜索。</span><span class="sxs-lookup"><span data-stu-id="20cb0-130">If the resource is not found, the search continues at the next level.</span></span>  
  
 <span data-ttu-id="20cb0-131">若要提高查找性能，可将 <xref:System.Resources.NeutralResourcesLanguageAttribute> 属性应用到主程序集，并向其传递将与主程序集一起使用的非特定语言的名称。</span><span class="sxs-lookup"><span data-stu-id="20cb0-131">To improve lookup performance, apply the <xref:System.Resources.NeutralResourcesLanguageAttribute> attribute to your main assembly, and pass it the name of the neutral language that will work with your main assembly.</span></span>  
  
> [!TIP]
>  <span data-ttu-id="20cb0-132">还可以使用 [ \<relativeBindForResources >](../../../docs/framework/configure-apps/file-schema/runtime/relativebindforresources-element.md) 配置元素来优化资源回退过程和运行时针对资源程序集探测所依据的。</span><span class="sxs-lookup"><span data-stu-id="20cb0-132">You may be able to use the [\<relativeBindForResources>](../../../docs/framework/configure-apps/file-schema/runtime/relativebindforresources-element.md) configuration element to optimize the resource fallback process and the process by which the runtime probes for resource assemblies.</span></span> <span data-ttu-id="20cb0-133">有关详细信息，请参阅 [优化资源回退进程](../../../docs/framework/resources/packaging-and-deploying-resources-in-desktop-apps.md#Optimizing)一节。</span><span class="sxs-lookup"><span data-stu-id="20cb0-133">For more information, see the [Optimizing the Resource Fallback Process](../../../docs/framework/resources/packaging-and-deploying-resources-in-desktop-apps.md#Optimizing) section.</span></span>  
  
 <span data-ttu-id="20cb0-134">资源回退进程包含以下步骤：</span><span class="sxs-lookup"><span data-stu-id="20cb0-134">The resource fallback process is involves the following steps:</span></span>  
  
1.  <span data-ttu-id="20cb0-135">运行时首先检查[全局程序集缓存](../../../docs/framework/app-domains/gac.md)，找到与为应用程序请求的区域性匹配的程序集。</span><span class="sxs-lookup"><span data-stu-id="20cb0-135">The runtime first checks the [global assembly cache](../../../docs/framework/app-domains/gac.md) for an assembly that matches the requested culture for your application.</span></span>  
  
     <span data-ttu-id="20cb0-136">全局程序集缓存可存储由许多应用程序共享的资源程序集。</span><span class="sxs-lookup"><span data-stu-id="20cb0-136">The global assembly cache can store resource assemblies that are shared by many applications.</span></span> <span data-ttu-id="20cb0-137">这使你无需在创建的每个应用程序的目录结构中包括特定的资源集。</span><span class="sxs-lookup"><span data-stu-id="20cb0-137">This frees you from having to include specific sets of resources in the directory structure of every application you create.</span></span> <span data-ttu-id="20cb0-138">如果运行时找到了对程序集的引用，它将搜索程序集，查找请求的资源。</span><span class="sxs-lookup"><span data-stu-id="20cb0-138">If the runtime finds a reference to the assembly, it searches the assembly for the requested resource.</span></span> <span data-ttu-id="20cb0-139">如果在程序集中找到了该项，则使用请求的资源。</span><span class="sxs-lookup"><span data-stu-id="20cb0-139">If it finds the entry in the assembly, it uses the requested resource.</span></span> <span data-ttu-id="20cb0-140">如果找不到该项，将继续搜索。</span><span class="sxs-lookup"><span data-stu-id="20cb0-140">If it doesn't find the entry, it continues the search.</span></span>  
  
2.  <span data-ttu-id="20cb0-141">接下来，运行时将检查当前正在执行程序集的目录，查找与请求的区域性匹配的目录。</span><span class="sxs-lookup"><span data-stu-id="20cb0-141">The runtime next checks the directory of the currently executing assembly for a directory that matches the requested culture.</span></span> <span data-ttu-id="20cb0-142">如果找到该目录，它将搜索该目录以找到请求的区域性的有效附属程序集。</span><span class="sxs-lookup"><span data-stu-id="20cb0-142">If it finds the directory, it searches that directory for a valid satellite assembly for the requested culture.</span></span> <span data-ttu-id="20cb0-143">然后，运行时将搜索附属程序集，寻找请求的资源。</span><span class="sxs-lookup"><span data-stu-id="20cb0-143">The runtime then searches the satellite assembly for the requested resource.</span></span> <span data-ttu-id="20cb0-144">如果在程序集中找到该资源，则使用它。</span><span class="sxs-lookup"><span data-stu-id="20cb0-144">If it finds the resource in the assembly, it uses it.</span></span> <span data-ttu-id="20cb0-145">如果找不到该资源，将继续搜索。</span><span class="sxs-lookup"><span data-stu-id="20cb0-145">If it doesn't find the resource, it continues the search.</span></span>
